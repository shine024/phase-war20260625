## ✅ 已修复：aura_data.gd 语法错误

### 错误信息
```
第 30 行：Expected expression for match pattern.
第 31 行：Expected ":" or "when" after "match" patterns.
```

### 问题原因
Godot 4.x 的 `match` 语句不支持逗号分隔的多行模式写法：
```gdscript
# ❌ 错误写法
match platform_type:
    2, 3, 7,  # TITAN, FORTRESS, SIEGE
    4, 8, 11,  # RADAR, CARRIER, OMEGA_PLATFORM
    12:  # COMMAND
        return true
```

### 修复方案
```gdscript
# ✅ 正确写法
match platform_type:
    2, 3, 7, 4, 8, 11, 12:  # TITAN, FORTRESS, SIEGE, RADAR, CARRIER, OMEGA_PLATFORM, COMMAND
        return true
    _:
        return false
```

### 修改文件
- `data/aura_data.gd` (行 24-31)

## ✅ 已修复：野战炮武器类型推断

### 问题发现

野战炮的 `combat_kind = 2` (SUPPORT)，射程 = 99，原始逻辑：
```gdscript
# 旧逻辑
if range >= 99:
    return GC.WeaponType.INDIRECT  # 正确，会被识别为曲射
```

但这个逻辑不够明确，SUPPORT 类型的火炮应该优先被识别为曲射。

### 修复方案

添加明确的 SUPPORT 类型判断：
```gdscript
# 新逻辑
# 支援单位（火炮/迫击炮等）→ INDIRECT（优先判定）
if combat_kind == 2:  # SUPPORT
    return GC.WeaponType.INDIRECT
# 曲射单位(range>=99)→ INDIRECT
if range >= 99:
    return GC.WeaponType.INDIRECT
```

## ✅ 已修复：SUPPORT 索敌方式

### 问题发现

野战炮 (SUPPORT) 类型没有明确的索敌方式映射，导致可能使用默认的 NEAREST_FIRST（从近到近）。

### 设定要求

根据设定文档：
- 炮类（SIEGE）和飞机（AIR）从远到近
- 其他单位从近到远

### 扩展设定

根据野战炮的实际定位，SUPPORT（火炮/迫击炮）和 FORT（固定工事）也应该从远到近索敌：
- ✅ LIGHT (轻装) → NEAREST_FIRST
- ✅ ARMOR (装甲) → NEAREST_FIRST
- ✅ SUPPORT (火炮/迫击炮) → FARTHEST_FIRST **(新增)**
- ✅ AIR (空中) → FARTHEST_FIRST
- ✅ FORT (堡垒/固定工事) → FARTHEST_FIRST **(新增)**

### 修复方案

添加索敌方式映射函数：
```gdscript
## 战斗类型到索敌方式的映射
static func get_targeting_mode_for_combat_kind(combat_kind: int) -> int:
    match combat_kind:
        CombatKind.SUPPORT:  # 支援（火炮、迫击炮）
            return TargetingMode.FARTHEST_FIRST
        CombatKind.AIR:     # 空中
            return TargetingMode.FARTHEST_FIRST
        CombatKind.FORT:    # 堡垒（固定防御工事）
            return TargetingMode.FARTHEST_FIRST
        _:  # LIGHT, ARMOR 默认从近到远
            return TargetingMode.NEAREST_FIRST
```

## 📝 代码审查发现

### 潜在问题：防御属性命名与实际用途不一致

在 `AttackCalculator.get_defense_vs()` 中：
```gdscript
## 注意：当前使用旧的属性名（defense_light/armor/air）但含义已改变
## defense_light 现在表示对直射防御（防DIRECT武器）
## defense_armor 现在表示对曲射防御（防INDIRECT武器）
## defense_air 现在表示对空射防御（防AERIAL武器）
```

这个注释说明：
- 属性名称（defense_light/armor/air）仍然使用"目标类型"命名
- 但实际用途已经改为"武器类型"防御
- 这容易造成混淆

**建议：** 按照修复计划，将属性重命名为：
- `defense_direct` (对直射防御)
- `defense_indirect` (对曲射防御)
- `defense_aerial` (对空射防御)

但这是一个大规模重构，需要在确保存档兼容性的前提下进行。
