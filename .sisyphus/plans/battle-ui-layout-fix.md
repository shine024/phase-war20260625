# 战斗界面布局修复计划

## 问题诊断

### 当前状态
- HUD 场景已创建但存在布局错乱
- 四个主要区域未正确定位
- 控件之间可能存在遮挡和覆盖

### 核心问题清单

#### 问题 1: 区域定位错乱
- **现象**：HealthBar、EnergyBar、PhaseInstrumentPanel、UnitInfoPanel 位置不合理
- **影响**：界面混乱，信息难以读取
- **严重程度**：高

#### 问题 2: 层级遮挡
- **现象**：背景可能遮挡 HUD 元素
- **影响**：关键信息不可见
- **严重程度**：高

#### 问题 3: 分辨率适配
- **现象**：不同分辨率下布局不一致
- **影响**：用户体验差
- **严重程度**：中

#### 问题 4: 数据绑定缺失
- **现象**：HUD 未连接到游戏管理器
- **影响**：信息不更新
- **严重程度**：高

---

## 修复策略

### 策略 1: 重构锚点布局
- 使用 CanvasLayer 确保 HUD 在最上层
- 为四个区域设置明确的锚点
- 使用 Container 确保响应式布局

### 策略 2: 统一层级管理
- 背景：Z-index -10
- HUD 元素：Z-index 100
- 确保不互相遮挡

### 策略 3: 响应式容器
- 使用 HBoxContainer / VBoxContainer 自动适应
- 设置合适的 margin 和 padding
- 避免硬编码位置

### 策略 4: 分阶段数据绑定
- 第一阶段：视觉布局修复
- 第二阶段：连接游戏管理器

---

## 详细执行步骤

### 第 1 步：诊断当前状态
- 打开 battle_hud.tscn
- 截图记录当前布局
- 标注所有问题区域

### 第 2 步：重构 HUD 场景
- 清理冗余节点
- 重新组织场景树结构
- 设置正确的节点关系

### 第 3 步：修复锚点布局
- 为 TopLeftContainer 设置左上锚点
- 为 TopRightPanel 设置右上锚点
- 为 BottomCenterPanel 设置底中锚点
- 为 TopCenterWave 设置顶中锚点

### 第 4 步：调整层级
- 确保 BattleHUD layer = 100
- 确保所有背景元素 layer = -10
- 验证 DamagePopupLayer layer = 200

### 第 5 步：响应式验证
- 测试 1280x720
- 测试 1920x1080
- 调整 padding 和 offset

### 第 6 步：数据绑定准备
- 在 battle_hud.gd 中预留绑定接口
- 编写调试日志
- 准备模拟数据

---

## 验收标准

### 布局正确性
- 四个区域各司其位，无重叠
- 在 1280x720 和 1920x1080 下正常显示
- 背景滚动不影响 HUD 显示

### 功能完整性
- HealthBar 和 EnergyBar 可以手动更新
- PhaseInstrument 显示槽位状态
- UnitInfoPanel 显示占位信息

### 性能要求
- FPS 保持稳定
- 无明显的 UI 渲染延迟

---

## 风险控制

### 识别的风险
- 可能破坏现有脚本逻辑
- 层级调整可能影响显示

### 缓解措施
- 逐步修改，每步验证
- 保留现有场景作为备份
- 使用 Git 追踪修改
