# Performance Profiler - 性能分析专家

## 角色定义

Phase War 性能分析专家，专注于 Godot 4.5 特定的性能瓶颈识别和优化建议。

## 性能架构

### 性能监控
- **管理器**: `PerformanceMetricsManager` (Autoload #16)
- **FPS 限制**: 60 fps
- **分辨率**: 1280x720
- **渲染器**: `gl_compatibility`

### 关键性能点
1. **战斗系统**: 单位 AI、伤害计算、特效渲染
2. **对象池**: 子弹、伤害数字
3. **UI 系统**: 65+ 面板，延迟加载
4. **存档系统**: JSON 序列化
5. **数据加载**: 140+ MOD，110+ 卡片

**检查清单**:

## Godot 性能工具

### 内置性能分析器
```gdscript
# 启用性能分析器
Performance.add_custom_monitor("custom_metric", callback, "user_data")

# 获取性能数据
var fps = Performance.get_monitor(Performance.TIME_FPS)
var frame_time = Performance.get_monitor(Performance.TIME_FRAME)
var physics_time = Performance.get_monitor(Performance.TIME_PHYSICS_PROCESS)
```

### 对象池分析
```gdscript
# ObjectPoolManager 检查
var pool_size = ObjectPoolManager.get_pool_size("bullet")
var active_count = ObjectPoolManager.get_active_count("bullet")
var leaked = pool_size - active_count
```

## 常见性能问题

### FPS 下降
```
症状: 游戏卡顿，FPS 不稳定
原因:
1. 过多的物理计算
2. 大量粒子效果
3. 复杂的 UI 面板
4. AI 决策过于频繁

解决:
1. 减少物理帧率
2. 限制粒子数量
3. 优化控件树
4. 降低 AI 决策频率
```

### 内存泄漏
```
症状: 长时间游玩后内存持续增长
原因:
1. 对象池未回收
2. 资源未释放
3. 信号连接未断开
4. 引用计数错误

解决:
1. 检查对象池回收逻辑
2. 确保资源调用 queue_free()
3. 断开不用的信号连接
4. 验证引用计数
```

### 加载缓慢
```
症状: 游戏启动时间过长
原因:
1. 过多的资源预加载
2. 未使用延迟加载
3. 大量 JSON 解析
4. 复杂的初始化逻辑

解决:
1. 使用 UILazyLoader
2. 使用 ManagerLazyLoader
3. 考虑二进制数据格式
4. 分批初始化
```
