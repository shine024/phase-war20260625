# Phase War 交互模式库

> **Version**: 1.0
> **Last Updated**: 2026-04-08
> **Purpose**: 定义Phase War UI的通用交互模式，作为所有后续UI设计的基础参考

## 文档说明

本文档定义了Phase War中使用的标准UI交互模式。所有新UI设计应优先使用这些模式，确保游戏界面的一致性和易用性。

**设计原则：**
- **键盘优先** - 所有功能必须可通过键盘完成
- **快速导航** - 支持方向键、Tab键、快捷键
- **清晰反馈** - 每个交互都有明确的视觉状态
- **无障碍友好** - 高对比度、大字体、颜色不作为唯一信息

## 1. 网格/列表导航模式

### 使用场景
- 蓝图收藏浏览（Blueprint Collection）
- 卡牌合成选择（Card Synthesis）
- 相位仪选择（Phase Instrument Selector）
- 任务列表浏览（Quest List）
- 成就查看（Achievement Panel）

### 交互方式

#### 键盘操作
```
↑↓         - 在项目之间上下移动
←→         - 在网格中左右移动
PageUp/Down - 快速翻页
Home/End    - 跳转到第一个/最后一个项目
Enter       - 选择/激活当前项目
Space       - 选择/激活当前项目（Enter的替代）
Escape      - 取消选择/返回上一级
Tab         - 移动到下一个交互区域
Shift+Tab   - 移动到上一个交互区域
```

#### 鼠标操作
```
单击        - 选择项目
双击        - 快速激活项目（选择+确认）
悬停        - 显示项目详细信息（tooltip）
拖拽        - 重新排序（如支持）
滚轮        - 滚动列表
```

#### 游戏手柄（未来支持）
```
D-Pad/左摇杆 - 导航项目
A/×         - 选择/激活
B/○         - 取消/返回
LT/RT       - 快速翻页
```

### 视觉状态

| 状态 | 视觉表现 | 颜色规范 | 动画效果 |
|------|---------|---------|---------|
| **正常** | 半透明背景，标准亮度 | `bg_color: Color(0.08, 0.10, 0.15, 0.92)` | 无 |
| **悬停** | 背景变亮，边框高亮 | `bg_color: Color(0.15, 0.20, 0.25, 0.95)`<br>`border_color: Color(0.4, 0.85, 1.0, 0.9)` | 淡入（100ms） |
| **选中** | 实心背景，高亮边框，左侧指示器 | `bg_color: Color(0.15, 0.25, 0.35, 0.95)`<br>`border_color: Color(0.4, 0.85, 1.0, 0.9)`<br>`左侧指示条: Color(0.3, 0.9, 0.5, 1.0)` | 淡入（150ms），轻微缩放（1.02×） |
| **禁用** | 半透明，去饱和 | `bg_color: Color(0.05, 0.05, 0.08, 0.6)`<br>`字体颜色: Color(0.4, 0.4, 0.45, 0.6)` | 无 |
| **当前装备** | ✓ 勾选标记，绿色强调 | `标记颜色: Color(0.3, 0.85, 0.5, 1.0)` | 无 |

### 无障碍考虑
- **焦点可见性** - 键盘焦点必须有清晰的视觉指示（边框或背景变化）
- **焦点追踪** - 焦点移动必须平滑，支持自动滚动以保持焦点可见
- **最小点击区域** - 每个项目最小点击区域 44×44px（WCAG 2.1 AAA标准）
- **键盘陷阱** - 不允许键盘陷阱，Escape必须始终能关闭/返回
- **屏幕阅读器** - 项目状态（选中/禁用/已装备）必须通过文本或属性暴露

### 实现示例
参考 `scenes/ui/phase_instrument_selector.gd` 中的列表导航实现。

## 2. 卡片选择和激活模式

### 使用场景
- 卡牌制造（Card Manufacturing）
- 卡牌合成（Card Synthesis）
- 卡牌强化（Card Enhancement）
- 法则选择（Phase Law Selection）

### 交互方式

#### 键盘操作
```
Space/Enter - 激活/取消激活当前卡片
数字键1-9   - 快速选择对应槽位的卡片
Delete/Backspace - 移除当前选中卡片
↑↓←→       - 在卡片网格中导航
```

#### 鼠标操作
```
单击        - 选择卡片（高亮显示）
再次单击    - 取消选择
拖拽        - 拖动卡片到槽位（如支持）
右键单击    - 显示上下文菜单（详细信息/移除）
悬停        - 显示卡片完整信息（tooltip）
```

### 视觉状态

| 状态 | 视觉表现 | 颜色规范 | 动画效果 |
|------|---------|---------|---------|
| **未选择** | 半透明，低亮度 | `透明度: 0.7`<br>`saturation: 0.6` | 无 |
| **悬停** | 亮度提升，轻微放大 | `亮度: 1.1`<br>`缩放: 1.05×` | 淡入（100ms）+ 缩放（100ms） |
| **已选择** | 高亮边框，向上浮动 | `边框: Color(0.4, 0.85, 1.0, 1.0)`<br>`阴影: offset(0, -8px)`<br>`位移:向上8px` | 向上浮动（200ms，easing: out-back） |
| **槽位激活** | 槽位发光，脉冲动画 | `槽位边框: Color(0.3, 0.9, 0.5, 1.0)`<br>`脉冲: 1.0→0.6→1.0 (1.5s)` | 持续脉冲动画 |
| **拖拽中** | 半透明跟随鼠标 | `透明度: 0.8`<br>`缩放: 0.9×` | 无 |

### 多选规则
- **单选模式** - 一次只能选择一张卡片（如：装备槽位）
- **多选模式** - 可以选择多张卡片（如：合成材料）
  - 最多选择数量：根据场景定义（通常2-5张）
  - 达到上限时：视觉提示（数量计数器变红），禁用未选择项

### 无障碍考虑
- **选择状态** - 必须有"已选择 X/Y"的文本提示
- **撤销操作** - Escape或再次单击可取消选择
- **错误提示** - 无效选择时显示明确原因（如"槽位已满"）
- **高对比度模式** - 边框宽度至少3px，高对比度颜色

### 实现示例
参考 `scenes/ui/card_synthesis_panel.gd` 中的卡片选择逻辑。

## 3. 进度条显示模式

### 使用场景
- 经验值进度（Blueprint Star Progress）
- 制造进度（Crafting Progress）
- 加载进度（Loading Screen）
- 波次进度（Battle Wave Progress）

### 交互方式

#### 键盘操作
```
Tab         - 焦点移动到进度条（如可交互）
Enter/Space - 查看详细信息（如可交互）
```

#### 鼠标操作
```
悬停        - 显示精确数值（如：753/1000 XP）
单击        - 查看详细信息（如可交互）
```

### 视觉设计

#### 进度条组成
```
┌────────────────────────────────────────┐
│ 背景槽（Background Track）              │
│ ┌──────────────────────┐ ── 填充条（Fill）│
│ ━━━━━━━━━━━━━━━━━━━━━● ── 进度指示器（Indicator）│
│        75%            │ ── 百分比文本（Percentage）│
│   750 / 1000 XP       │ ── 精确数值（Exact Value）│
└────────────────────────────────────────┘
```

#### 颜色规范

| 进度类型 | 填充颜色 | 里程碑颜色 | 背景颜色 |
|---------|---------|-----------|---------|
| **经验/升级** | `Color(0.3, 0.7, 1.0, 1.0)` (蓝色) | `Color(0.4, 0.9, 0.5, 1.0)` (绿色) | `Color(0.1, 0.12, 0.16, 0.9)` |
| **制造/合成** | `Color(0.95, 0.75, 0.35, 1.0)` (金色) | `Color(0.4, 0.9, 0.5, 1.0)` (绿色) | `Color(0.1, 0.12, 0.16, 0.9)` |
| **战斗波次** | `Color(0.9, 0.3, 0.3, 1.0)` (红色) | `Color(0.95, 0.85, 0.2, 1.0)` (黄色) | `Color(0.1, 0.12, 0.16, 0.9)` |
| **加载进度** | `Color(0.6, 0.5, 0.9, 1.0)` (紫色) | N/A | `Color(0.1, 0.12, 0.16, 0.9)` |

#### 动画效果

| 事件 | 动画 | 时长 | Easing |
|------|------|------|--------|
| **进度增加** | 填充条平滑增长 | 300ms | `out-quart` |
| **达到里程碑** | 星星闪烁 + 粒子爆发 | 600ms | `out-back` |
| **进度满** | 光芒扫过 + 脉冲 | 800ms | `in-out-sine` |
| **进度减少** | 填充条平滑缩减（不常见） | 200ms | `in-quart` |

#### 文本显示规则
- **默认显示** - 百分比（如：75%）
- **悬停显示** - 精确数值（如：750 / 1000 XP）
- **里程碑标记** - 在进度条上显示关键点（如：星级升级点）
- **完成状态** - 显示"完成！"或"MAX"徽章

### 无障碍考虑
- **颜色独立性** - 不能仅依赖颜色传达进度，必须配合文本或图标
- **屏幕阅读器** - 进度值必须通过`min`, `max`, `value`属性或文本暴露
- **高对比度** - 填充条与背景槽对比度至少 4.5:1
- **文字大小** - 百分比文本最小 14px，精确数值最小 12px
- **动画开关** - 提供选项关闭动画（减少前庭刺激）

### 参考实现
参考 Slay the Spire 的 XP 条设计：
- 平滑的进度增长动画
- 达到阈值时的庆祝效果
- 悬停显示精确数值

## 4. 信息面板展开/折叠模式

### 使用场景
- 卡牌详细信息（Card Details Tooltip）
- 蓝图属性面板（Blueprint Stats Panel）
- 任务详细描述（Quest Description）
- 成就详情（Achievement Details）

### 交互方式

#### 键盘操作
```
Space/Enter - 展开/折叠当前焦点项
F1         - 显示帮助/详细信息（全局快捷键）
Escape     - 关闭展开的面板
↑↓         - 在展开项内部滚动（如适用）
```

#### 鼠标操作
```
悬停        - 显示简略tooltip（延迟500ms）
单击        - 展开/折叠详细信息
双击        - 快速展开并锁定（不自动关闭）
拖拽        - 拖动面板位置（如可移动）
```

### 视觉状态

| 状态 | 视觉表现 | 动画效果 |
|------|---------|---------|
| **折叠** | 仅显示标题行 + 展开/折叠图标 | 无 |
| **展开中** | 面板从标题行向下展开 | 高度增长（250ms，easing: out-quart） |
| **展开** | 显示完整内容，关闭按钮可见 | 淡入（200ms） |
| **折叠中** | 面板向上收缩 | 高度缩减（200ms，easing: in-quart） |
| **锁定** - | 不自动关闭，显示锁定图标 | 无 |

### 布局模式

#### 模式A：内联展开（Inline Expansion）
```
┌─────────────────────────────────────┐
│ ▶ 卡牌名称 | 稀有度 | 星级          │  ← 折叠状态
├─────────────────────────────────────┤
│ ▼ 卡牌名称 | 稀有度 | 星级          │  ← 展开状态
│   ┌─────────────────────────────┐  │
│   │ 详细属性...                 │  │
│   │ 描述文本...                 │  │
│   └─────────────────────────────┘  │
└─────────────────────────────────────┘
```

#### 模式B：浮动面板（Floating Panel）
```
┌─────────────────┐
│ 卡牌名称         │ ← 源项目（始终可见）
└────────┬────────┘
         │
         ▼
┌─────────────────────────────────┐
│ 卡牌详细信息           [×]      │ ← 浮动面板
├─────────────────────────────────┤
│ 属性列表...                     │
│ 描述文本...                     │
│ [关闭]                         │
└─────────────────────────────────┘
```

#### 模式C：侧边滑出（Side Panel）
```
┌───────────────┬───────────────────┐
│ 主内容区域     │ ▶ 详细信息        │ ← 折叠状态
│               ├───────────────────┤
│               │ ▼ 详细信息        │ ← 展开状态
│               │ ┌───────────────┐ │
│               │ │ 详细属性...   │ │
│               │ │ 描述...       │ │
│               │ └───────────────┘ │
└───────────────┴───────────────────┘
```

### 自动关闭规则
- **悬停tooltip** - 鼠标离开后延迟 500ms 自动关闭
- **单击展开** - 需要手动关闭（单击关闭按钮或按Escape）
- **键盘展开** - 需要手动关闭（按Escape或Space）
- **拖拽面板** - 拖拽后自动锁定，不自动关闭

### 无障碍考虑
- **焦点管理** - 展开时焦点移动到面板内容，关闭时返回源项目
- **Escape支持** - 所有展开面板必须响应Escape关闭
- **屏幕阅读器** - 展开/折叠状态必须通过`aria-expanded`属性暴露
- **键盘陷阱** - 不允许键盘陷阱，Tab键必须能进出面板
- **最小尺寸** - 展开面板最小尺寸 300×200px，避免内容过小

### 实现示例
参考 `scenes/ui/card_affix_tooltip.gd` 中的tooltip实现。

## 5. 确认对话框模式

### 使用场景
- 重要操作确认（如：删除蓝图、消耗纳米材料）
- 战斗结束结算（Battle Result）
- 系统消息通知（System Notification）
- 错误提示（Error Alert）

### 交互方式

#### 键盘操作
```
Enter       - 确认/执行操作（默认焦点）
Escape      - 取消/关闭对话框
Tab         - 在按钮之间切换焦点
Space       - 激活当前焦点按钮
```

#### 鼠标操作
```
单击        - 点击按钮执行对应操作
```

### 对话框类型

#### 类型A：确认/取消（Confirmation）
```
┌─────────────────────────────────────┐
│ ⚠ 确认操作                    [×]   │
├─────────────────────────────────────┤
│                                     │
│  确定要消耗 50 纳米材料升级此蓝图？  │
│                                     │
│  当前碎片：120/150                   │
│  升级后：★★ → ★★★                   │
│                                     │
├─────────────────────────────────────┤
│              [取消]  [确认升级]      │
└─────────────────────────────────────┘
```

**规则：**
- 主操作按钮（确认）放在右侧，高亮显示
- 次要操作按钮（取消）放在左侧，标准样式
- 默认焦点在取消按钮（危险操作）或确认按钮（安全操作）
- Enter键激活默认焦点按钮

#### 类型B：通知（Notification）
```
┌─────────────────────────────────────┐
│ ℹ 蓝图已解锁                [×]    │
├─────────────────────────────────────┤
│                                     │
│  新蓝图：泰坦机甲 MK2               │
│  稀有度：史诗                       │
│  初始星级：★★★                     │
│                                     │
│  [查看详情]                         │
│             [确定]                  │
└─────────────────────────────────────┘
```

**规则：**
- 单个"确定"按钮，居中或右对齐
- 可选的"查看详情"按钮（如适用）
- 默认焦点在"确定"按钮
- 支持按Enter或Space关闭

#### 类型C：错误（Error）
```
┌─────────────────────────────────────┐
│ ✕ 无法执行操作               [×]   │
├─────────────────────────────────────┤
│                                     │
│  纳米材料不足！                     │
│                                     │
│  需要：50 纳米                      │
│  当前：35 纳米                       │
│  缺少：15 纳米                       │
│                                     │
├─────────────────────────────────────┤
│                  [确定]             │
└─────────────────────────────────────┘
```

**规则：**
- 红色警告图标和边框
- 清晰的错误原因说明
- 建设性的解决方案（如：缺少的资源、如何获取）
- 单个"确定"按钮，默认焦点

### 视觉规范

| 对话框类型 | 边框颜色 | 背景颜色 | 图标 |
|-----------|---------|---------|------|
| **确认** | `Color(0.4, 0.85, 1.0, 1.0)` (蓝色) | `Color(0.08, 0.10, 0.15, 0.98)` | ⚠ (U+26A0) |
| **通知** | `Color(0.3, 0.9, 0.5, 1.0)` (绿色) | `Color(0.08, 0.10, 0.15, 0.98)` | ℹ (U+2139) |
| **错误** | `Color(0.9, 0.3, 0.3, 1.0)` (红色) | `Color(0.15, 0.05, 0.05, 0.98)` | ✕ (U+2715) |

### 动画效果
- **出现** - 从中心淡入 + 轻微缩放（0.9× → 1.0×，200ms）
- **消失** - 向中心淡出 + 缩放（1.0× → 0.9×，150ms）
- **按钮悬停** - 背景变亮 + 轻微向上位移（100ms）

### 无障碍考虑
- **焦点陷阱** - 对话框打开时，焦点必须移动到对话框内，关闭后返回触发元素
- **默认焦点** - 危险操作默认焦点在"取消"，安全操作默认焦点在"确认"
- **Escape支持** - 所有对话框必须响应Escape关闭（等同于取消/确定）
- **标题层级** - 对话框标题必须是 `<h2>` 或更高层级，便于屏幕阅读器
- **按钮顺序** - 按钮顺序必须与视觉顺序一致（Tab键顺序）

### 实现示例
参考 Godot 的 `AcceptDialog` 节点，已在 `phase_instrument_selector.gd` 中使用。

## 6. 滚动和分页模式

### 使用场景
- 大量项目浏览（Blueprint Collection > 50 items）
- 任务列表（Quest List）
- 成就列表（Achievement List）
- 排行榜（Leaderboard）

### 交互方式

#### 键盘操作
```
↑↓         - 在项目之间导航
PageUp/Down - 滚动一页（约80%视口高度）
Home/End    - 跳转到顶部/底部
Space/Enter - 激活当前项目
```

#### 鼠标操作
```
滚轮        - 垂直滚动（默认滚动3行/滚轮单位）
拖拽滚动条  - 拖动滚动条滑块快速导航
单击滚动条  - 跳转到对应位置
```

### 模式选择

#### 模式A：连续滚动（Infinite Scroll）
**适用场景** - 项目数量 < 100，加载成本较低

```
┌─────────────────────────────────────┐
│ ▶ 项目1                             │
│ ▶ 项目2                             │
│ ▶ 项目3                             │ ← 视口底部
│ ▶ 项目4 (滚动中加载)                │
│ ▶ 项目5                             │
└─────────────────────────────────────┘
      │                    │
      ▼                    ▼
  [滚动条]            [加载指示器]
```

**规则：**
- 显示所有项目，无分页
- 滚动到底部时自动加载更多（如适用）
- 滚动条显示相对位置
- 保持滚动位置（返回时恢复）

#### 模式B：分页浏览（Pagination）
**适用场景** - 项目数量 > 100，或需要明确范围

```
┌─────────────────────────────────────┐
│ 第 3 / 10 页                        │
├─────────────────────────────────────┤
│ ▶ 项目21                            │
│ ▶ 项目22                            │
│ ▶ 项目23                            │
│ ▶ 项目24                            │
│ ▶ 项目25                            │
└─────────────────────────────────────┘
         [◀ 上一页]  [1] [2] [3] [4] [5]  [下一页 ▶]
```

**规则：**
- 每页固定项目数量（建议 20-50）
- 显示当前页/总页数
- 提供页码跳转（1-5页直接显示，更多用"..."）
- 上一页/下一页按钮
- 可选：跳转到指定页（输入框）

#### 模式C：字母索引（Alphabetical Index）
**适用场景** - 按名称排序的项目列表

```
┌─────────────────────────────────────┐
│ [A] [B] [C] [D] ... [X] [Y] [Z]     │ ← 字母索引栏
├─────────────────────────────────────┤
│ ▶ C - 炮塔平台                       │
│ ▶ C - 冲锋枪                         │
│ ▶ C - 重型机甲                       │
│ ▶ C - 磁暴线圈                       │
└─────────────────────────────────────┘
```

**规则：**
- 顶部字母索引栏（可点击快速跳转）
- 高亮当前视图的字母范围
- 无项目字母禁用（灰色显示）
- 支持键盘输入字母快速跳转

### 滚动行为规范

| 滚动类型 | 滚动量 | 动画时长 | Easing |
|---------|--------|---------|--------|
| **键盘方向键** | 1个项目高度 | 100ms | `out-quad` |
| **PageUp/Down** | 视口高度的80% | 150ms | `out-quad` |
| **鼠标滚轮** | 3个项目高度 | 无（即时） | N/A |
| **滚动条拖拽** | 跟随鼠标 | 无（即时） | N/A |
| **Home/End** | 到顶部/底部 | 200ms | `in-out-quart` |
| **搜索跳转** | 到目标项目 | 250ms | `out-quart` |

### 性能考虑
- **虚拟滚动** - 项目数量 > 200 时使用虚拟滚动（仅渲染可见项）
- **延迟加载** - 分页模式下按需加载（不预加载所有页）
- **滚动位置缓存** - 记住每个列表的滚动位置（返回时恢复）

### 无障碍考虑
- **滚动速度** - 提供选项调整滚动速度（慢/中/快）
- **动画开关** - 提供选项关闭滚动动画（减少前庭刺激）
- **焦点可见** - 滚动后焦点项目必须完全可见（自动调整）
- **屏幕阅读器** - 提供"跳转到第X页"或"显示X-Y项"的快捷命令
- **最小触摸区域** - 移动端（如支持）滚动条最小宽度 8px

### 实现示例
参考 Godot 的 `ScrollContainer` 节点，已在多个UI面板中使用。

| 快捷键 | 功能 | 优先级 |

| `Space` | 暂停/继续 | 高 |
| `1-9` | 快速选择对应法则 | 高 |
| `Tab` | 切换下一个单位 | 中 |
| `Ctrl` | 显示详细信息（按住） | 低 |

#### 上下文快捷键（战前准备）
| 快捷键 | 功能 | 优先级 |

| `A` | 打开/关闭成就 | 低 |

#### UI中的快捷键显示
```
┌─────────────────────────────────────┐

├─────────────────────────────────────┤

│ ▶ 项目2          (2)               │
│ ▶ 项目3          (3)               │
└─────────────────────────────────────┘
```

**规则：**
- 快捷键提示用括号包裹，灰色显示
- 放在对应按钮/标签的右侧
- 仅显示当前上下文有效的快捷键
- 支持的快捷键在首次使用时高亮提示（教程模式）

### 快捷键冲突处理
1. **优先级定义** - 全局 > 上下文 > 可重绑定
2. **冲突检测** - 绑定新快捷键时检测冲突，显示警告

4. **配置文件** - 快捷键绑定保存在用户配置文件（`settings_manager.gd`）

### 无障碍考虑
- **可重绑定** - 所有快捷键必须可重新绑定（除Escape和Ctrl+S）
- **单手操作** - 提供单手可达的快捷键选项
- **语音输入** - 快捷键触发必须可通过语音命令替代（如支持）
- **帮助文档** - 提供完整的快捷键列表文档（可从游戏内访问）

## 8. 工具提示（Tooltip）模式

### 使用场景
- 卡牌详细信息（Card Details）

- 词汇表术语（Glossary Terms）
- 交互提示（Interaction Hints）

### 交互方式

#### 鼠标操作
```
悬停        - 延迟500ms后显示tooltip
移动        - tooltip跟随鼠标移动（可选）
离开        - 延迟300ms后隐藏tooltip
```

#### 键盘操作
```
焦点+长按Space - 显示tooltip（延迟800ms）
焦点+F1      - 立即显示tooltip（无延迟）
焦点+Escape  - 隐藏tooltip
```

### Tooltip类型

#### 类型A：纯文本（Text Only）
```
┌─────────────────────────────────────┐
│ 攻击力                              │
├─────────────────────────────────────┤
│ 此单位的物理攻击伤害输出             │
└─────────────────────────────────────┘
```

#### 类型B：富文本（Rich Text）
```
┌─────────────────────────────────────┐
│ 攻击力                              │
├─────────────────────────────────────┤
│ 此单位的[color=red]物理攻击[/color]  │
│ 伤害输出，受[color=gold]力量[/color] │
│ 属性影响                            │
└─────────────────────────────────────┘
```

#### 类型C：结构化信息（Structured）
```
┌─────────────────────────────────────┐
│ ★★★ 泰坦机甲 MK2                    │
├─────────────────────────────────────┤
│ 稀有度：史诗                         │
│ 类型：平台                           │
│                                     │
│ 属性：                              │
│ • 生命值：500 (+150)                │
│ • 攻击力：45 (+12)                  │
│ • 防御力：20 (+5)                   │
│                                     │
│ 特殊效果：                          │
│ 重装甲：受到的伤害减少20%            │
└─────────────────────────────────────┘
```

#### 类型D：交互式（Interactive - 未来支持）
```
┌─────────────────────────────────────┐
│ 攻击力                      [?]     │ ← 点击查看详情
├─────────────────────────────────────┤
│ 此单位的物理攻击伤害输出             │
│                                     │
│ [点击查看完整计算公式]               │ ← 可点击链接
└─────────────────────────────────────┘
```

### 显示规则

| 规则 | 说明 |
|------|------|
| **位置** | 默认显示在目标元素下方，避免超出屏幕边界 |
| **最大宽度** | 300px（超出自动换行） |
| **最大高度** | 400px（超出显示滚动条） |
| **z-index** | 最高层级，覆盖所有其他UI |
| **背景** | 半透明深色背景 `Color(0.05, 0.05, 0.08, 0.98)` |
| **边框** | 1px 实线 `Color(0.4, 0.85, 1.0, 0.8)` |
| **圆角** | 6px |
| **阴影** | `offset(0, 4px), blur(8px), Color(0, 0, 0, 0.5)` |

### 动画效果
- **出现** - 淡入 + 轻微向下位移（-4px → 0px，150ms）
- **消失** - 淡出（100ms）
- **跟随鼠标** - 无动画（即时更新位置）

### 延迟配置
| 场景 | 延迟时间 | 原因 |
|------|---------|------|
| **默认** | 500ms | 避免误触发 |
| **教程模式** | 200ms | 帮助新手探索 |
| **键盘触发** | 800ms | 给予足够时间 |
| **F1立即显示** | 0ms | 用户主动请求 |

### 无障碍考虑
- **键盘访问** - 必须可通过键盘触发（长按Space或F1）
- **屏幕阅读器** - Tooltip内容必须通过`aria-label`或`title`属性暴露
- **高对比度** - Tooltip背景与前景对比度至少 7:1
- **动画开关** - 提供选项禁用动画（减少前庭刺激）
- **字体大小** - 最小 12px，支持缩放至 16px

### 实现示例
参考 `scenes/ui/card_affix_tooltip.gd` 中的tooltip实现。

## 9. 拖拽操作模式（Drag & Drop Pattern）

### 使用场景
- 卡牌到槽位（Card to Slot）
- 物品整理（Inventory Organization）
- 技能栏配置（Skill Bar Setup - 未来）

### 交互方式

#### 鼠标操作
```
按下左键    - 开始拖拽（延迟150ms，避免误触发）
拖拽        - 移动时显示半透明ghost跟随鼠标
释放左键    - 放置到目标位置或取消
```

#### 键盘操作
```
Enter/Space - 选择项目
↑↓←→       - 移动到目标位置
Enter       - 确认放置
Escape      - 取消拖拽
```

### 视觉状态

| 状态 | 视觉表现 | 动画效果 |
|------|---------|---------|
| **可拖拽** | 鼠标悬停时显示"抓取"光标 | 光标变化（即时） |
| **拖拽中** | 原项目半透明，ghost元素跟随鼠标 | 淡出（100ms） |
| **有效目标** | 目标槽位高亮显示 | 脉冲动画（500ms循环） |
| **无效目标** | 目标槽位显示红色禁止图标 | 无 |
| **放置成功** | Ghost元素消失，目标槽位显示新项目 | 缩放（1.1× → 1.0×，200ms） |
| **放置失败** | Ghost元素飞回原位置 | 飞回动画（300ms，easing: out-back） |

### 拖拽Ghost规范
```
┌─────────────────────────────────────┐
│ [半透明ghost]                       │
│ ★★★ 泰坦机甲 MK2                    │
│ 稀有度：史诗                         │
└─────────────────────────────────────┘
      ↓
  跟随鼠标移动
  透明度：0.7
  缩放：0.9×
  偏移：鼠标位置右下方8px
```

### 放置规则

| 场景 | 行为 | 反馈 |
|------|------|------|
| **有效槽位** | 放置成功，交换或合并 | ✓ 绿色勾选动画 + 音效 |
| **无效槽位** | 飞回原位置，错误提示 | ✕ 红色叉号 + 错误音 |
| **空区域** | 飞回原位置 | 无特殊反馈 |
| **相同项目** | 交换位置 | 交换动画（300ms） |

### 键盘替代流程
```
1. Tab键移动焦点到源项目
2. Enter/Space选择（项目高亮）
3. 方向键移动到目标槽位
4. Enter确认放置
5. 或Escape取消
```

### 无障碍考虑
- **键盘必需** - 所有拖拽操作必须可通过键盘完成
- **拖拽延迟** - 鼠标按下150ms后才开始拖拽，避免误触发
- **取消操作** - Escape键或拖到无效区域自动取消
- **运动敏感度** - 提供选项调整拖拽ghost的运动（减少/禁用动画）
- **屏幕阅读器** - 拖拽状态必须通过文本提示（如："已选中泰坦机甲，按Enter放置"）

### 实现示例
参考 `scenes/ui/custom_drag_card_item.gd` 中的拖拽实现。

## 10. 搜索和过滤模式（Search & Filter Pattern）

### 使用场景
- 蓝图搜索（Blueprint Search）
- 卡牌过滤（Card Filter）
- 任务筛选（Quest Filter）

### 交互方式

#### 键盘操作
```
Ctrl+F     - 焦点移动到搜索框（全局快捷键）
输入字符   - 实时过滤结果
↑↓        - 在过滤结果中导航
Escape    - 清空搜索/关闭搜索框
```

#### 鼠标操作
```
单击搜索框 - 焦点移动到搜索框
输入字符   - 实时过滤结果
单击过滤器 - 切换过滤条件
```

### 搜索框设计

```
┌─────────────────────────────────────┐
│ 🔍 搜索蓝图...            [×]      │ ← 搜索框
├─────────────────────────────────────┤
│ 过滤器：                            │
│ [全部] [平台] [武器] [法则]         │ ← 快速过滤器
│ 稀有度：                            │
│ [☑] 普通 [☑] 稀有 [☐] 史诗         │ ← 多选过滤器
├─────────────────────────────────────┤
│ ▶ C - 冲锋枪                        │ ← 过滤结果
│ ▶ C - 磁暴线圈                      │
└─────────────────────────────────────┘
```

### 搜索类型

#### 类型A：实时搜索（Real-time Search）
- **触发** - 每输入一个字符立即更新结果
- **延迟** - 300ms防抖，避免频繁更新
- **适用** - 项目数量 < 500

#### 类型B：按回车搜索（Enter to Search）
- **触发** - 输入完成后按Enter或点击搜索按钮
- **延迟** - 无延迟，手动触发
- **适用** - 项目数量 > 500 或性能敏感场景

#### 类型C：自动补全（Autocomplete）
```
┌─────────────────────────────────────┐
│ 🔍 泰坦...                          │
├─────────────────────────────────────┤
│ • 泰坦机甲 MK1                      │ ← 自动补全建议
│ • 泰坦机甲 MK2                      │
│ • 泰坦重炮                          │
└─────────────────────────────────────┘
```

### 过滤器类型

#### 单选过滤器（Radio Filter）
```
[全部] [平台] [武器] [法则]
```
- 一次只能选择一个
- 选中的过滤器高亮显示

#### 多选过滤器（Checkbox Filter）
```
稀有度：[☑] 普通 [☑] 稀有 [☐] 史诗
```
- 可以选择多个条件（OR逻辑）
- 至少选择一个条件

#### 范围过滤器（Range Filter）
```
星级：★★ ━━━━━●━━━━ ███████★ (3-9★)
```
- 滑块或输入框
- 支持最小/最大值

### 搜索反馈

| 场景 | 反馈 | 视觉表现 |
|------|------|---------|
| **有结果** | 显示匹配的项目数量 | "找到 12 个蓝图" |
| **无结果** | 显示空状态提示 | "未找到匹配的蓝图，尝试其他关键词" |
| **搜索中** | 显示加载指示器 | 小型加载spinner |
| **搜索错误** | 显示错误提示 | "搜索失败：网络错误" |

### 高亮匹配
```
┌─────────────────────────────────────┐
│ ▶ [机甲] 泰坦机甲 MK2               │ ← 关键词高亮
│ ▶ 机枪重炮台                        │
│ ▶ 机甲核心                          │
└─────────────────────────────────────┘
```

**规则：**
- 匹配的字符用黄色背景高亮 `Color(0.95, 0.85, 0.2, 0.4)`
- 不区分大小写
- 支持部分匹配（如："机" 匹配 "机甲"）

### 无障碍考虑
- **标签关联** - 搜索框必须有对应的`<label>`元素
- **清除按钮** - 提供快速清除按钮（[×]），键盘可访问
- **结果计数** - 必须通过文本或`aria-live`区域 announce 结果数量
- **过滤器状态** - 过滤器状态必须清晰可见（如：选中/未选中）
- **空状态提示** - 无结果时提供建设性建议（如：清除过滤器、尝试其他关键词）

## 附录E：设计决策记录（ADR）

### ADR-001: 键盘优先设计
**日期**: 2026-04-08
**状态**: 已批准

**决策**：Phase War UI采用键盘优先设计原则。

**理由**：
- PC平台主要输入方式是键盘/鼠标
- 策略游戏需要频繁导航和选择，键盘效率更高
- 无障碍要求：所有功能必须可仅用键盘完成
- 为未来游戏手柄支持预留接口

**影响**：
- 所有交互模式必须定义完整的键盘操作
- 焦点管理系统需要优先实现
- UI组件必须支持键盘导航

### ADR-002: 颜色无障碍规范
**日期**: 2026-04-08
**状态**: 已批准

**决策**：颜色不能作为唯一的信息传达方式。

**理由**：
- 支持**色盲**玩家（约8%男性，0.5%女性）
- 符合WCAG 2.1 AAA标准
- 高对比度模式支持

**影响**：
- 所有状态必须有非颜色信息（图标、文本、边框）
- 进度条、选择状态等必须配合文本或图标
- 提供高对比度模式选项

### ADR-003: 动画可配置
**日期**: 2026-04-08
**状态**: 已批准

**决策**：所有UI动画必须可关闭或减速。

**理由**：
- 减少**前庭刺激**（motion sensitivity）
- 提升性能（低端设备）
- 用户偏好尊重

**影响**：
- 动画时长配置化（快速/标准/慢速）
- 提供全局动画开关
- 默认使用标准时长，但保留快速选项
