# 《相位战争：构装纪元》完整游戏介绍

> 本文档面向玩家与协作者，概括世界观、玩法循环、主要系统与内容规模。技术细节与实现以仓库内各专题文档为准。

---

## 一、游戏是什么

**《相位战争：构装纪元》**（英文名：**Phase War**，工程内标识 `phase-war`）是一款基于 **Godot 4.5** 的 **横版自动对战** 策略游戏。

你扮演掌控 **相位仪** 的指挥官，通过装配 **构装卡**（平台、武器、能量与法则）定义战场上自动作战的武器平台；敌方从另一侧持续压境，双方在射程内自动交火。你的目标是在保护己方 **相位场核心（驱动器）** 的前提下，清空既定波次的敌人并取得胜利。

**一句话**：用有限的准备能量与四个相位仪槽位，搭配出能扛能打、还能用法则扭转局面的构装组合，在五个战争时代、百关推进中成长。

---

## 二、世界观与题材

- **时代跨度**：从一战到近未来的架空战争史，强调「构装」与「相位技术」对战争的改写。
- **叙事基调**：硬核军事装备感（坦克、装甲车、机枪、导弹、光束武器等）与科幻元素（相位法则、高能平台）并存。
- **玩家角色**：通过相位仪远程投射构装单位，而非亲自操控单兵；战斗以 **自动移动 + 自动开火** 为主，侧重 **战前构筑** 与 **资源/法则时机**。

---

## 三、核心游戏循环

1. **战前准备**  
   整理背包、将平台/武器/部分能量卡与法则装入相位仪（受准备阶段能量与槽位限制）；可进行合成、蓝图相关操作、任务与商店等（以当前版本 UI 为准）。

2. **进入战斗**  
   我方单位按规则从左侧生成并向右推进；敌方按关卡配置从右侧波次刷新。战斗中有能量自然增减，可配合主动法则改变局面。

3. **胜负结算**  
   胜利可获得基础资源、蓝图碎片、纳米材料等成长要素；失败则通常返回准备或主界面重新调整构筑。

4. **长期推进**  
   共 **100 关**，分为 **5 个时代**（每时代约 20 关）：一战、二战、冷战、现代、近未来。时代越靠后，敌人波次、数量规模与掉落倾向通常越强。

该循环在 `GameManager` 中以 **准备 → 战斗 → 战后** 阶段管理，与 `BattleManager` 的刷新与胜负判定衔接。

## 四、战前准备：背包与相位仪

### 背包中的卡牌类型

- **平台卡**：决定单位的底盘形态、耐久、移速与武器挂载能力（如侦察车、主战坦克、固定炮、母舰、全装机动舱等）。
- **武器卡**：决定伤害、射程、射速与输出风格（冲锋枪、步枪、机枪、迫击炮/火箭、狙击、防空、光束、导弹、粒子炮等）。
- **能量卡**  
  - **战前能量**：提高战斗开始时的初始能量。  
  - **能量收集**：提高战斗中每秒自然回复。  
  - **即时能量**：在背包中使用可立刻获得能量（使用后消耗该卡），并可能与战中回复存在取舍。
- **合成卡**：由 **平台 + 武器** 在合成界面合并而成，在相位仪中占用槽位但代表一套固定组合，便于快速装配。

### 相位仪（四个槽位）

- 将卡牌从背包 **拖入** 相位仪槽位完成装备；装备会消耗 **准备阶段能量**（与每张卡的能量消耗相关）。
- 常见配法：**平台 + 武器** 成对构成一种「前线构装」；也可使用已合成的 **合成卡** 节省操作。
- 部分能量卡可装入相位仪以改变开局与持续回复；**相位法则**（见下文）需在满足研究值、环境条件等前提下装备与激活。

## 五、战斗阶段：自动战、刷新与能量

### 单位刷新（典型规则，以当前关卡与版本为准）

- **我方**：周期性从已装备的 **平台+武器组合**（或合成卡所代表的组合）中 **随机** 选取一种生成单位；同侧场上单位数存在 **上限**（如 5）。
- **敌方**：按关卡所属时代与配置，以 **波次** 形式刷新，波次间隔与总波次数随关卡与时代变化。

### 自动战斗

- 双方单位相向移动，进入射程后自动攻击；部分平台/武器/法则附带护盾、灼烧、控制、链式伤害等特殊机制。
- **词条（Affix）** 会在单位生成时修改 **生命、移速、伤害、射程、攻速、减伤、暴击、吸血、溅射** 等属性，并在命中等环节触发战斗向效果（详见仓库内词条文档）。

### 战斗中的能量

- 战斗内有 **基础自然回复** 与 **相位仪基础持续消耗**；战前装入的能量卡会叠加修正 **初始能量** 与 **每秒回复**。
- 能量常用于 **主动相位法则** 的施放（具体消耗与冷却以各法则定义为准）。

### 胜负条件

- **胜利**：在保护 **相位场驱动器** 的前提下，完成敌方波次并消灭场上敌方单位（具体判定与关卡特殊规则以 `BattleManager` / 战场逻辑为准）。  
- **失败**：**相位场驱动器被摧毁** 即告失败。

## 六、关卡与敌人

- **关卡结构**：100 关，5 时代；`LevelEras` 等数据驱动每关的波次数、每波单位数、间隔等。
- **敌人原型**：从一战士兵班、盾卫车、迫击炮小队，到二战坦克与狙击塔、冷战导弹车，再到头目级超重型坦克、光束炮舰、风暴核心等；另有大量 **按时代生成的常规敌人** 填充波次。
- **敌方掉落**：击败敌人可滚动 **蓝图碎片** 等掉落，与敌人原型上的 `drops` 配置相关；胜利后还有关卡结算向的基础资源与额外蓝图机会（见 `GameManager` / `BattleManager` 与资源表）。

---

## 七、成长系统：蓝图、合成、解析与资源

### 蓝图与碎片

- 蓝图对应可解锁的平台、武器或能量向内容；通过战斗掉落、任务、商店等途径积累 **碎片** 与 **纳米材料**。
- **蓝图解析**（`BlueprintAnalysisManager`）：消耗碎片与时间可提升蓝图等级，支持多蓝图并行解析与存档恢复。
- **蓝图管理**（`BlueprintManager`）：记录解锁状态、等级与材料，并通过信号驱动 UI 更新。

### 合成（`SynthesisManager`）

- **卡牌 + 卡牌**：提升稀有度（如 common → uncommon → rare → mythic），有成功率与递增的纳米材料消耗。  
- **卡牌 + 蓝图**：融合生成带蓝图加成的强化卡，成功率与消耗单独配置。

### 基础资源

- 如 **基本纳米颗粒**、**能量块** 等，由关卡与时代相关的掉落表发放，用于制造、解析或与经济系统联动。

### 制造与信息界面

- 准备界面提供 **制造区域** 入口（装配卡、解析卡、蓝图库、信息库等）；部分子功能在持续迭代中，以当前场景与脚本实现为准（参见 `docs/manufacture_and_drop.md`）。

## 八、相位法则

法则来自 **钢铁、烈焰、雷霆、虚空** 等家族，分为 **被动** 与 **主动**：

- **被动示例**：相位装甲、快速维修、热能过载（灼烧）等，改变友军生存或输出特性。  
- **主动示例**：堡垒之墙、前线炮击、电磁风暴、链式放电、时空涟漪、护盾转移等，提供护盾、范围伤害、控制与时间类效果。

每条法则可有 **研究需求** 与 **环境条件**（天气、地形、能量场、昼夜等），需在战前满足方可激活。完整列表与 ID 可参考 `data/phase_laws.gd` 与 `blueprint_law_full_table.md`。

---

## 九、法则碎片与卡牌强化（扩展系统）

- **LawShardManager**：与法则碎片收集、使用相关的进度与规则衔接 UI/存档（随版本扩展）。  
- **CardEnhancementManager**：用于卡牌强化流程，与蓝图、材料或专用道具联动。  
具体入口与数值以游戏内当前版本为准。

## 十、词条（Affix）系统

- 构装卡可携带多条 **词条**，区分稀有度与等级，部分词条可 **变异** 或 **锁定**。  
- 词条在 **单位生成** 时写入属性，在 **弹道命中、击杀** 等环节由战斗处理器结算暴击、穿甲、溅射、吸血等效果。  
详细列表见 `AFFIX_DOCUMENTATION.md`、`AFFIX_QUICK_REFERENCE.md` 与 `docs/AFFIX_COMBAT_EFFECTS.md`。

---

## 十一、特殊遭遇：相位师对战

- 进入战斗时有一定概率（如 **15%**）遭遇 **相位师**：使用排行榜或内嵌 NPC 数据中的相位师配置，优先匹配 **当前关卡所属势力** 的防守方相位师。  
- 相位师战会替换或追加敌方配置（如 `BattleManager` 中 `_spawn_phase_master_units`），胜利/失败在 `GameManager` 中有专门分支处理。

---

## 十二、任务与其他

- **QuestManager** 驱动任务定义与进度（任务数据见 `data/quest_definitions.gd`）。  
- **FactionSystemManager** 处理势力相关逻辑，与关卡势力、相位师遭遇等关联。  
- **AudioManager** 负责 BGM/SFX；**SaveManager** 负责存档读写。

根据仓库内 **`blueprint_law_full_table.md`** 等导出表（随数据迭代可能变化）：

## 十四、如何运行与工程入口

- 使用 **Godot 4.5** 打开项目根目录下的 `project.godot`，按 **F5** 运行。  
- **启动主场景**（`project.godot` 配置）：`res://scenes/title_screen.tscn`；主流程与战斗常由 `res://scenes/main.tscn` 与 `main.gd` 组织。  
- 分辨率示例：1280×720；渲染可为 GL 兼容模式以便多端一致。
