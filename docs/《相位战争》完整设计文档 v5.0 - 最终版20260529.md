## 一、核心设计原则

| 原则             | 说明                               |
| ---------------- | ---------------------------------- |
| **自动对战**     | 玩家布阵，系统自动战斗，战中无操作 |
| **策略在战前**   | 所有决策在编队界面完成             |
| **克制决定胜负** | 攻击/防御的多维克制是核心          |
| **成长有方向**   | 进化分支让玩家选择"转职"方向       |
| **改造可继承**   | 进化后改造完全保留，可替换         |

## 四、武器类型系统

### 4.1 武器类型枚举

```gdscript
enum WeaponType {
    DIRECT = 0,      # 直射：坦克炮、步枪、机枪、反坦克炮
    INDIRECT = 1,    # 曲射：迫击炮、榴弹炮、火箭炮
    AERIAL = 2       # 空射：战斗机、攻击机、无人机
}
```

### 4.2 各类型核心规则

| 属性         | 直射(DIRECT)             | 曲射(INDIRECT)         | 空射(AERIAL)           |
| ------------ | ------------------------ | ---------------------- | ---------------------- |
| **选敌逻辑** | 距离最近的敌人           | 优先被克制的类型       | 优先被克制的类型       |
| **射程衰减** | 有（距离越远伤害越低）   | 无                     | 无                     |
| **射程限制** | 有限（range字段，2-5格） | 无限（全图）           | 无限（全图）           |
| **典型单位** | 坦克、步兵、反坦克炮     | 迫击炮、榴弹炮、火箭炮 | 战斗机、攻击机、无人机 |
| **可被拦截** | 否                       | 否                     | 是（由防空单位）       |

### 4.3 选敌逻辑详细规则

#### 直射(DIRECT)
```
目标选择优先级：
1. 距离最近的敌人（欧几里得距离）
2. 同距离时，选择当前生命值最低的敌人
3. 同距离同生命时，选择部署时间最早的

特殊：射程内无敌人时，向敌方基地方向移动
```

#### 曲射(INDIRECT)
```
目标选择优先级：
1. 优先选择【被本单位的攻击类型克制】的敌人
   - 如果 attack_armor 高 → 优先选装甲类敌人
   - 如果 attack_light 高 → 优先选轻装类敌人
   - 如果 attack_air 高 → 优先选空中类敌人
2. 无克制目标时，选择距离最近的敌人
3. 同距离时，选择生命值最低的

特殊：不移动（部署速度通常为0或极低），全图攻击
```

#### 空射(AERIAL)
```
目标选择优先级：
1. 优先选择【空中单位】（空对空）
2. 无空中目标时，选择被地面攻击类型克制的敌人
3. 无克制目标时，选择距离最近的

特殊：飞行单位，可被防空单位攻击
```

### 4.4 射程衰减公式（仅直射）

```gdscript
# 伤害衰减率 = 1 - (距离 / 最大射程) × 衰减系数

func calculate_damage_attenuation(distance, max_range, attenuation_factor = 0.5):
    if distance <= 0:
        return 1.0
    if distance >= max_range:
        return 1.0 - attenuation_factor
    
    var t = distance / max_range
    return 1.0 - (t * attenuation_factor)
```

### 4.5 衰减系数表

| 武器类型    | 衰减系数 | 最远衰减 | 示例         |
| ----------- | -------- | -------- | ------------ |
| 冲锋枪/SMG  | 0.6      | -60%     | MP18、汤普森 |
| 步枪/RIFLE  | 0.4      | -40%     | 毛瑟、加兰德 |
| 机枪/MG     | 0.5      | -50%     | MG42、M60    |
| 坦克炮/TANK | 0.3      | -30%     | 虎王、M1A2   |
| 反坦克炮/AT | 0.2      | -20%     | 铁拳、标枪   |
| 狙击/SNIPER | 0.1      | -10%     | 特殊单位     |

## 五、战斗系统核心规则

### 5.1 核心伤害规则

```gdscript
# 规则1：攻击力 ≤ 对应防御时，伤害为0（无法击穿）
if attack_value <= defense_value:
    final_damage = 0
    return 0

# 规则2：攻击力 > 防御时，计算减免后伤害
if attack_value > defense_value:
    final_damage = attack_value × (100 / (100 + defense_value))
```

### 5.2 完整伤害计算流程

```gdscript
# 1. 获取基础攻击值（根据目标类型）
var base_damage = get_attack_value(attacker, target)
# attack_light / attack_armor / attack_air

# 2. 检查是否击穿防御
if base_damage <= defense_value:
    return 0

# 3. 应用射程衰减（仅直射）
var attenuation = calculate_attenuation(distance, range, weapon_type)
base_damage = base_damage × attenuation

# 4. 应用防御减免
var defense = get_defense_value(target, attacker)
final_damage = base_damage × (100 / (100 + defense))

# 5. 应用改造/模块化词条加成
final_damage = final_damage × mod_bonus × module_slot_bonus
```

### 5.3 攻击速度与攻击间隔

```gdscript
# 每种攻击类型有独立的攻击速度参数
# 攻击周期 = 1.0 / attack_speed
# 攻击周期 = 前摇 + 动作 + 后摇

attack_cycle = 1.0 / attack_speed
attack_cooldown = attack_cycle - attack_windup - attack_active
```

### 5.4 部署速度含义

| 部署速度 | 含义     | 到达战场时间（基准） | 典型单位           |
| -------- | -------- | -------------------- | ------------------ |
| 0        | 瞬间部署 | 0秒                  | 固定炮台、要塞     |
| 1        | 极慢     | 10秒                 | 重型坦克、重型火炮 |
| 2        | 慢       | 8秒                  | 虎王、巨神机甲     |
| 3        | 中慢     | 6秒                  | 主战坦克、中型坦克 |
| 4        | 中速     | 4秒                  | 步兵、步战车       |
| 5        | 快速     | 2.5秒                | 轻型坦克、反坦克车 |
| 6        | 极快     | 1.5秒                | 侦察车、无人机     |
| 7        | 瞬间进场 | 0.5秒                | 空降兵、空投单位   |

**公式**：`到达时间(秒) = (8 - deploy_speed) × 1.5`（deploy_speed ≤ 7时）

### 10.4 改造替换规则

```gdscript
func replace_mod(unit, old_mod_id, new_mod_id):
    # 检查新改造类型是否与已有改造冲突
    var conflict = false
    for mod_id in unit.mods:
        if get_mod_type(mod_id) == get_mod_type(new_mod_id):
            conflict = true
            unit.mods.erase(mod_id)
            unit.mods.append(new_mod_id)
            break
    
    if not conflict and unit.mods.size() < 9:
        unit.mods.append(new_mod_id)
    
    recalculate_power(unit)
```
