Phase War 美术资源生成工作流
版本: v3.0 — 2026-06-06
适用范围: 势力专属卡、单位卡美术资源生成
核心目标: 画风统一、背景规范、质量达标、流程可复用

---

流程总览

1. 生成前检查
2. 批量生成（1024x1024，C方案风格）
3. 自动化后处理（绿底检测、抠图）
4. 人工视觉审核
5. 问题卡重绘
6. 部署
7. 游戏内验证

---

第1步：生成前检查

1.1 确认目标列表
- 读取 enemy_unit_manifest.gd 的 E段/F段/池子卡定义
- 列出每张卡需要的视觉文件名、卡名、势力、战斗类型
- 对照 assets/card_icons/units/ 现有文件，确认哪些已存在、哪些缺失
- 记录需要重绘/重生成的卡及原因

1.2 确认API可用性
- 检查 ~/.hermes/config.yaml 中的 API key 是否完整
- 确认 Base URL: https://apihub.agnes-ai.com/v1
- 确认 Model: agnes-image-2.0-flash
- 发一张测试图验证 API 连通性和返回质量

1.3 确认生成参数
- 分辨率: 1024x1024 (1:1 正方形)
- 输出目录: docs/美术资源预览_重做/
- 部署目录: assets/card_icons/units/

1.4 确认Prompt结构

每张卡使用以下统一prompt结构：

Fictional military unit/card "{card_name}", {专属描述},
STRICTLY flat pure side profile silhouette facing left, head turned exactly 90 degrees,
ONLY side view visible, zero frontal perspective, flat 2D side view,
STRICT FULL BODY - HEAD, TORSO, UPPER LEGS, LOWER LEGS, BOOTS, COMPLETE from top to ground,
{装备/武器/载具描述},
matte painting style, detailed weathering and surface texture,
complex dramatic lighting with rim light from top-left,
solid chroma key green #00FF00 background ONLY,
centered composition with full body fitting in frame,
NO realistic human faces, NO real-world military insignia, NO text, NO watermark, 1024x1024

Prompt关键模块说明：
- {card_name}: 卡的中文名称
- {专属描述}: 每张卡独特的描述（阵营特征、装备、武器、载具细节等）
- STRICTLY flat pure side profile silhouette facing left: 强制严格正侧面，左侧
- STRICT FULL BODY: 强制全身，从头到脚
- matte painting style, detailed weathering: C方案风格，写实概念艺术风
- solid chroma key green #00FF00 background ONLY: 强制纯绿底
- NO realistic human faces...: 禁止项

---

第2步：批量生成

2.1 分批生成
- 每批 5-7 张，间隔 3 秒
- 每张卡使用其专属 description 生成，不共用 prompt
- 所有图片保存为 {id}_预览.png 格式

2.2 生成后立即下载
- 每张图返回 URL 后立即下载，防止网络中断丢失
- 记录每张图片的基本信息：prompt 原文、尺寸、文件大小、临时路径

2.3 风格指南（C方案）
- 避免卡通化关键词: "clean geometric shapes", "bold silhouette", "hard surface design", "game card art"
- 使用写实概念艺术: "matte painting", "detailed weathering", "surface texture"
- 光照: "complex dramatic lighting", "rim light from top-left"
- 质感: "weathered", "rust streaks", "scuff marks", "oil stains", "chipped paint"

---

第3步：自动化后处理

3.1 抠图（绿底转透明）
使用 Pillow 执行 chroma key 抠图：
- 将图片转为 RGBA 模式
- 遍历每个像素，满足以下条件的标记为背景：
  - G > 160 AND R < 140 AND B < 140 AND G > R + 60 AND G > B + 60
  - 边缘羽化: G > 200 AND R < 180 AND B < 180 → alpha = 255 - ((G-200)/55 * 255)
- 背景像素设为 (0,0,0,0) 完全透明
- 保存为 PNG 格式（无损压缩）

3.2 抠图质量检测
- 计算透明像素占比（应 > 30%，否则可能背景不纯绿）
- 采样图片四角确认背景颜色
- 如果透明像素占比 < 10%，标记为"绿底异常"，需要检查原图

---

第4步：人工视觉审核

4.1 审核清单（每张卡逐张检查）
- [ ] 背景是否为纯绿 #00FF00？
- [ ] 视角是否为严格正侧面（非 3/4）？
- [ ] 是否为完整全身（从头到脚，包括腿和靴子）？
- [ ] 抠图边缘是否干净（无白边/杂色/锯齿）？
- [ ] 主体是否居中？
- [ ] 画风是否符合 C方案（写实概念艺术，非卡通）？
- [ ] 是否有禁止元素（人脸/现实军徽/文字/水印）？
- [ ] 整体质量是否达标（清晰、细节丰富）？

4.2 审核标准
- 通过: 8 项全部通过
- 警告: 7 项通过，1 项轻微问题（如绿底偏亮偏暗但不影响使用）
- 失败: 任意关键项未通过（视角错误、缺主体、画风不符、抠图破损）→ 必须重绘

---

第5步：问题卡重绘

5.1 重绘策略
- 失败的卡: 重新生成，用更严格的 prompt
- 警告的卡: 先给人类审核，确认是否可接受
- 重绘时记录原因，用于优化 prompt

5.2 重绘prompt增强
根据失败原因追加负面词：
- 如果是 3/4 视角: "STRICTLY flat pure side profile silhouette, head turned exactly 90 degrees, ONLY side view visible, zero frontal perspective"
- 如果不是全身: "STRICT FULL BODY - HEAD, TORSO, UPPER LEGS, LOWER LEGS, BOOTS, COMPLETE from top to ground"
- 如果是卡通风: "matte painting style, detailed weathering, surface texture"（去掉硬表面/几何关键词）
- 如果背景不是绿底: "solid chroma key green #00FF00 background ONLY, no gradients, no lighting on background"

---

第6步：部署

6.1 文件部署
- 将最终通过审核的图放到 assets/card_icons/units/
- 文件名按 vis_player_NNN.png 或 vis_enemy_NNN.png 规范
- 1024x1024 分辨率，透明背景

6.2 代码映射检查
- 确认 enemy_unit_manifest.gd 中的 visual_id 映射正确
- 确认 card_id → visual_id 的映射链完整

---

第7步：游戏内验证

7.1 Godot 刷新
- 提醒用户按 Ctrl+Shift+R 刷新资源
- 确认没有脚本错误

7.2 运行时检查
- 进入游戏，打开背包查看卡面
- 进入战场查看单位显示
- 确认所有卡都有正确图片，没有几何色块占位

---

常见错误与对策

1. 视角偏 3/4: prompt 中多次强调 "STRICTLY flat pure side profile", "zero frontal perspective", "flat 2D side view"
2. 主体不完整: prompt 中明确列出 "STRICT FULL BODY - HEAD, TORSO, UPPER LEGS, LOWER LEGS, BOOTS"
3. 卡通感: 使用 "matte painting", "detailed weathering" 替代 "game card art", "hard surface"
4. 绿底不纯: 抠图时先采样确认背景颜色，透明度 < 30% 时需要检查原图
5. 绿底在图片非角落: 如果四角采样不到绿底，用全图绿色像素平均法
6. 重复生成风格不一: 使用固定 prompt 模板，只替换 {card_name} 和 {专属描述}

自动化脚本参考

API 配置提取:
- config.yaml 中的 model.api_key 和 model.base_url
- 拼接 POST URL: {base_url}/images/generations
- Headers: Authorization: Bearer {api_key}, Content-Type: application/json

生成请求 Payload:
{
  "model": "agnes-image-2.0-flash",
  "prompt": "{prompt_text}",
  "image_size": "1024x1024"
}

抠图阈值:
- 严格: G > 160 AND R < 140 AND B < 140 AND G > R + 60 AND G > B + 60
- 边缘羽化: G > 200 AND R < 180 AND B < 180 → alpha = 255 - ((G-200)/55 * 255)
