## Phase 2: 战斗系统重写 [核心玩法]

### 2.1 三种选敌逻辑 — 新文件 `scripts/battle/target_selection.gd`

```gdscript
enum TargetMode { DIRECT, INDIRECT, AERIAL }

## 直射: 距离最近 → 同距最低HP → 同距同HP最早部署
## 超出射程时向敌方基地方向移动
func select_target_direct(attacker, enemies) -> Node

## 曲射: 优先被克制类型 → 无克制则最近 → 同距最低HP
## 不移动
func select_target_indirect(attacker, enemies) -> Node

## 空射: 优先空中 → 无空中则克制目标 → 最近
func select_target_aerial(attacker, enemies) -> Node
```

克制优先判断逻辑:

- attack_light最高 → 优先选LIGHT类型
- attack_armor最高 → 优先选ARMOR类型
- attack_air最高 → 优先选AIR类型

### 2.2 射程衰减 — 新文件 `scripts/battle/damage_attenuation.gd`

```gdscript
## 仅直射有衰减,曲射和空射无衰减
## 伤害衰减率 = 1 - (距离/最大射程) × 衰减系数
## 距离=0 → 100%, 距离=max_range → (1-系数)%

const ATTENUATION_FACTORS = {
    "SMG":    0.6,   # -60%
    "RIFLE":  0.4,   # -40%
    "MG":     0.5,   # -50%
    "TANK":   0.3,   # -30%
    "AT":     0.2,   # -20%
    "SNIPER": 0.1    # -10%
}

func calculate_attenuation(distance, max_range, weapon_sub_type) -> float
```

需要为每个单位分配weapon_sub_type(SMG/RIFLE/MG/TANK/AT/SNIPER)。可通过unit_class和attack值推断:

- LIGHT+range<=2 → SMG
- LIGHT+range==3 → RIFLE
- SUPPORT+weapon==DIRECT+对空>0 → SNIPER (或特殊标记)
- ARMOR → TANK
- 轻装+对甲高 → AT
- 支援+射速高 → MG

### 2.3 伤害公式重写

```gdscript
## 完整伤害计算流程
func calculate_damage(attacker, target, distance) -> float:
    # 1. 根据目标类型选攻击值
    var base_damage = get_attack_vs(attacker, target.unit_class)
    # attack_light/armor/air 对应目标类型

    # 2. 击穿检查: attack <= defense → 伤害=0
    if base_damage <= get_defense(target, attacker.unit_class):
        return 0.0

    # 3. 射程衰减(仅直射)
    if attacker.weapon_type == DIRECT:
        base_damage *= calculate_attenuation(distance, attacker.range, ...)

    # 4. 防御减免: damage × 100/(100+defense)
    var def = get_defense(target, attacker.unit_class)
    var final_damage = base_damage * (100.0 / (100.0 + def))

    # 5. 改造加成
    final_damage *= get_mod_damage_multiplier(attacker.mods)

    # 6. 强化加成(百分比)
    final_damage *= (1.0 + attacker.enhance_level * 0.05)  # Lv1=+5%, Lv10=+60%

    return final_damage
```

```gdscript
func replace_mod(unit, old_mod_id, new_mod_id):
    # 检查新改造类型是否与已有改造冲突
    var conflict = false
    for mod_id in unit.mods:
        if get_mod_type(mod_id) == get_mod_type(new_mod_id):
            conflict = true
            unit.mods.erase(mod_id)
            unit.mods.append(new_mod_id)
            break

        unit.mods.append(new_mod_id)

    recalculate_power(unit)
```

## 执行顺序与依赖关系

```
Phase 1 (数据结构)
    ↓
Phase 2 (战斗系统) ──────────────────────┐
    ↓                                    ↓
Phase 3 (养成系统: 强化+改造+品质)    可并行
    ↓                                    ↓
Phase 4 (进化系统) ──────────────────────┘
    ↓
Phase 5 (情报手册)
    ↓
Phase 6 (清理旧系统)
```

- Phase 1 是一切基础,必须最先完成
- Phase 2 是核心体验,优先级最高
- Phase 3 和 Phase 4 可以在 Phase 2 完成后并行开发
- Phase 5 依赖 Phase 4 (进化需要情报100%条件)
- Phase 6 最后做,避免中途破坏兼容性
