### 4.2 缴获卡构建逻辑

缴获卡 `captured_<archetype_id>` 由 `CapturedUnitCards` 构建，逻辑如下：

1. **静态表优先**: `CapturedCardStats.get_stats(drop_id)` 命中则直接取值
2. **动态 Fallback**: 若静态表未命中，从 `EnemyArchetypes.get_config(archetype_id)` 读取后：
   - 有 `attack_damage > 0`：按 `weapon_type` 分配到三维攻击
   - 无 `attack_damage`：直接读 `attack_light/armor/air`
   - defense 三维：读 `defense_light/armor/air`，缺则 0
   - `base_speed = abs(speed) / 0.65`（speed 为负值时）
   - `power = round(hp*0.3 + Σattack*2 + Σdefense*1.5 + base_speed*0.1)`
