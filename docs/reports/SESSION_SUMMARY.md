## 🏗️ 系统架构

### 游戏流程集成
```
任务完成 → 奖励包含势力声望
     ↓
QuestManager._grant_rewards()
     ↓
FactionSystemManager.add_faction_reputation()
     ↓
发出信号 faction_reputation_changed
     ↓
UI面板实时更新
     ↓
SaveManager 保存数据到 save.json
```

### 三大核心系统

#### 1. 势力系统 (FactionSystemManager)
- 7个独立势力
- 声望 0-10000 的量化体系
- 等级 1-10 的进度曲线
- 独立的商店库存系统
- 🎯 **用途**: 扩展游戏深度，增加重复可玩性

#### 2. 卡牌强化系统 (CardEnhancementManager)
- 10级递进强化
- 成功率风险设计 (95% → 40%)
- 成本指数增长 (100 → 3000纳米)
- 属性加成 5%-60%
- 🎯 **用途**: 提供长期目标，鼓励玩家投入

#### 3. 关卡信息系统 (LevelInformation)
- 100个关卡完整设定
- 5个时代背景故事
- 环境系统 (天气、地形、能量场、时间)
- 难度倍数动态调整
- 🎯 **用途**: 强化故事感和沉浸感

## 🔄 数据流向

### 存储架构
```
save.json
├─ blueprint          (蓝图数据)
├─ company            (公司数据)
├─ basic_resources    (基础资源)
├─ phase_law          (法则数据)
├─ quest              (任务数据)
├─ faction_system     ✅ 新增
│  ├─ faction_reputation    (各势力声望)
│  ├─ faction_level         (各势力等级)
│  └─ faction_store_inventory (商店库存)
├─ card_enhancement   ✅ 新增
│  └─ card_enhancement_level (卡牌强化等级)
├─ phase_slots        (法则槽位)
└─ backpack_extra_ids (背包卡牌)
```

### 信号通信
```
FactionSystemManager                CardEnhancementManager
├─ faction_reputation_changed        ├─ enhancement_started
├─ faction_level_up                  ├─ enhancement_completed
└─ faction_store_updated             └─ enhancement_failed
         ↑                                   ↑
         │ (连接到)                         │ (连接到)
         │                                   │
    FactionPanel                  CardEnhancementPanel
```
