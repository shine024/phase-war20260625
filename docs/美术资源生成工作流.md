\n# Phase War 美术资源生成工作流

## 概述

本工作流定义了从需求确认到最终部署的完整美术资源生成流程，针对 Phase War 卡牌游戏\n
的资源生产规范。核心目标：确保每张卡面的画风统一、背景规范、质量达标。\n

## 工作流版本\nv2.0 — 2026-06-06\n\n
---\n\n
## 流程总览\n\n
1. 生成前检查\n2. 批量生成\n3. 自动化后处理\n4. 人工视觉审核\n5. 问题卡重绘\n6. 部署\n7. 游戏内验证\n\n
---\n\n
## 第1步：生成前检查\n\n
### 1.1 确认目标列表\n- 读取 enemy_unit_manifest.gd 的 E段/F段/池子卡定义\n- 列出每张卡需要的视觉文件名、卡名、势力、战斗类型\n- 对照 assets/card_icons/units/ 现有文件，确认哪些已存在、哪些缺失\n\n
### 1.2 确认API可用性\n- 检查 config.yaml 中的 API key 是否完整（非截断）\n- 发一张测试图验证 API 连通性和返回质量\n- 确认代理设置（http://127.0.0.1:10808）\n\n
### 1.3 确认生成Prompt模板\n每张卡使用统一prompt结构：\n\n
```
Fictional sci-fi military unit "{card_name}", {era_description},\nbold silhouette, high contrast, muted metallic palette with 1-2 accent colors,\nclean geometric shapes, hard surface design,\nSTRICT true side profile, full body, centered composition,\nsolid chroma key green #00FF00 background ONLY, no shadows, no ground,\nNO realistic human faces, NO real-world military insignia, NO text, NO watermark,\ngame card art style, readable at 28px thumbnail.\n```\n\n
关键强制词：\n- "bold silhouette, high contrast" — 大剪影高对比，适应 28×28 小图\n- "muted metallic palette with 1-2 accent colors" — 金属低饱和 + 1-2 个点缀色\n- "clean geometric shapes, hard surface design" — 硬表面几何风格，不是写实渲染\n- "STRICT true side profile" — 严格正侧面，不是 3/4 视角\n- "solid chroma key green #00FF00 background ONLY" — 强制绿底\n- "NO realistic human faces, NO real-world military insignia" — 禁止人脸和现实军徽\n- "game card art style, readable at 28px thumbnail" — 卡牌风格，小尺寸可读\n\n
---\n\n
## 第2步：批量生成\n\n
### 2.1 分批生成\n- 每批 5-7 张，间隔 10 秒\n- 每张卡使用其专属 description 生成，不共用prompt\n\n
### 2.2 生成后立即下载\n- 每张图返回URL后立即下载，防止网络中断丢失\n- 记录每张大图的基本信息：prompt原文、尺寸、文件大小、临时路径\n\n
---\n\n
## 第3步：自动化后处理\n\n
### 3.1 绿底检测与修复
自动脚本执行以下检查：\n\n
a) 背景纯度检测\n- 采样图片四角各 50×50 像素区域\n- 计算平均值颜色\n- 如果平均值不是绿色（不在 #00FF00 ±20 范围内），进入绿底修复流程\n\n
b) 绿底修复流程\n- 用 Pillow 打开图片\n- 识别前景（非绿色区域），用边缘检测或 flood fill\n- 将背景区域统一填为 #00FF00\n- 前景边缘做羽化过渡\n\n
c) 抠图质量检测\n- 计算边缘像素的 alpha 通道平滑度\n- 如果边缘有杂色/白边/锯齿 → 标记为"抠图差"\n\n
d) 主体完整性检测\n- 查找非背景色像素的边界框\n- 检查边界框是否覆盖图片高度的 60% 以上\n- 检查主体是否居中\n\n
### 3.2 输出处理报告
每张卡输出一行：\n
```\nvis_player_036: OK (绿底#00FF00, 主体占比82%, 边缘平滑)\nvis_player_039: WARN (绿底偏#10FF10, 主体占比75%, 边缘有白边)\nvis_player_041: FAIL (背景#FFFFFF非绿色, 主体占比45%, 边缘锯齿严重)\n```\n\n
---\n\n
## 第4步：人工视觉审核\n\n
### 4.1 审核清单（每张卡逐张检查）
- [ ] 视角是否为严格正侧面？（不是 3/4 视角）\n- [ ] 是否为全身？（头到脚完整）\n- [ ] 背景是否为纯绿 #00FF00？\n- [ ] 抠图边缘是否干净？（无白边/杂色/锯齿）\n- [ ] 主体是否居中？\n- [ ] 方向是否正确？（我方朝右，敌方朝左）\n- [ ] 画风是否符合要求？（硬表面科幻剪影风，不是写实摄影）\n- [ ] 是否有禁止元素？（人脸/现实军徽/文字/水印）\n\n
### 4.2 审核标准\n- **通过**：8项全部通过\n- **警告**：7项通过，1项轻微问题（如绿底偏亮偏暗但不影响使用）\n- **失败**：任意关键项未通过（视角错误、缺主体、画风不符、抠图破损）→ 必须重绘\n\n
---\n\n
## 第5步：问题卡重绘\n\n
### 5.1 重绘策略\n- 失败的卡：重新生成，用更严格的 prompt\n- 警告的卡：先给人类审核，确认是否可接受\n- 重绘时记录原因，用于优化prompt\n\n
### 5.2 重绘prompt增强\n根据失败原因追加负面词：\n- 如果是 3/4 视角："absolute pure side profile, no perspective, 90 degree profile view"\n- 如果是写实风："cartoon game card style, not photorealistic, no skin texture, no fabric detail"\n- 如果背景不是绿底："flat #00FF00 background only, no gradients, no lighting on background"\n- 如果抠图差："clean edge cutout, no white halo, no artifacts, crisp silhouette"\n\n
---\n\n
## 第6步：部署\n\n
### 6.1 文件部署\n- 将最终通过审核的图放到 assets/card_icons/units/\n- 文件名按 vis_player_NNN.png 或 vis_enemy_NNN.png 规范（1:1 正方形，512×512 或 640×640）\n- 同步生成 .import 文件（Godot 纹理导入配置）\n\n
### 6.2 代码映射检查\n- 确认 enemy_unit_manifest.gd 中的 visual_id 映射正确\n- 确认 card_id → visual_id 的映射链完整\n\n
---\n\n
## 第7步：游戏内验证\n\n
### 7.1 Godot 刷新\n- 提醒用户按 Ctrl+Shift+R 刷新资源\n- 确认没有脚本错误\n\n
### 7.2 运行时检查\n- 进入游戏，打开背包查看卡面\n- 进入战场查看单位显示\n- 确认所有卡都有正确图片，没有几何色块占位\n\n
---\n\n
## 技术规格总结\n\n| 项 | 值 |\n|---|---|\n| 画布比例 | 1:1 正方形 |\n| 推荐分辨率 | 512×512 或 640×640 |\n| 背景颜色 | 纯绿 #00FF00 (RGB 0,255,0) |\n| 画风 | 硬表面科幻 + 大剪影 + 高对比 + 低饱和金属色 |\n| 视角 | 严格正侧面 |\n| 禁止 | 人脸、现实军徽、文字、水印、绿底用于主体 |\n| 部署路径 | assets/card_icons/units/\n\n
