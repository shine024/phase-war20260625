# 战场敌人卡图引用错误诊断记录

**日期**：2026-06-15
**状态**：待修复（已定位根因与修复点，未改动代码）
**相关文档**：`docs/游戏单位清单_v3.md`

---

## 一、问题现象

战场上的 **36 种固定敌人**（C段：一战~近未来的步兵/精英/BOSS）显示的是「缴获后翻新版」的卡面图（`vis_player_036~071`），而非敌人自己的原图（`vis_enemy_036~071`）。

> 例：一战基础步兵 `enemy_ww1_infantry_basic` 在战场上显示的是 `vis_player_036.png`（缴获卡面），而非 `vis_enemy_036.png`（敌人原图）。

这与 `游戏单位清单_v3.md` 的 **C段**（战场敌人 visual_id = `vis_enemy_036~071`）不符。

---

## 二、根因

**同一个 enemy_id 有两个视觉身份**，共用了一个 `visual_id` 字段，导致张冠李戴：

| 身份 | 应显示 | 文档段落 |
|------|--------|---------|
| 战场敌人 sprite | `vis_enemy_036~071`（敌人原图） | C段（三、C段固定原型） |
| 玩家缴获后的卡牌图标 | `vis_player_036~071`（缴获卡面） | C'/C''段（缴获卡面） |

而 manifest 的 `visual_id` 字段对 C段敌人被统一设成了 `vis_player`（缴获卡面），战场 sprite 加载又读了这个字段 → 战场敌人显示缴获卡面。

---

## 三、错误数据流（完整调用链）

### A. 战场 sprite（错误路径）
```
scenes/units/enemy_unit.gd:296
  └─ EnemyArchetypes.resolve_card_icon_texture_path(archetype_id, merged_cfg, lookup_id)
scenes/units/construct_unit.gd:690
  └─ EnemyArchetypes.resolve_card_icon_texture_path(...)   # 玩家方用 enemy archetype 视觉时
     └─ data/enemy_archetypes.gd:447  resolve_card_icon_texture_path
        步骤2 → EnemyUnitManifest.get_unit_icon_path_for_archetype(archetype_id)
           └─ data/enemy_unit_manifest.gd:395
              └─ _unit_icon_by_archetype[arch] = manifest row 的 visual_id
                 └─ manifest row visual_id 由 _make_fixed_row(:519) → _visual_id_for_source_id(:790) 生成
                    └─ _visual_id_for_source_id 对 FIXED_ENEMY_IDS 返回
                       "vis_player_%03d" % (36 + fixed_idx)   ← 错误：缴获卡面
```

### B. 卡牌图标（背包/UI，正确路径）
```
scripts/ui_asset_loader.gd:239  card_icon_path_for(c)
  └─ _manifest_icon_for_archetype(arch)   (:191)
     └─ EnemyUnitManifest.get_unit_icon_path_for_archetype(arch)   → visual_id = vis_player（缴获卡面，正确）
```

> 注意：卡牌图标走 `card_icon_path_for`，**不经过** `resolve_card_icon_texture_path`，所以它用 `vis_player` 是对的。问题只在战场 sprite 那条链。

## 四、素材文件实际情况（均存在，无缺失）

```
assets/card_icons/units/vis_enemy_036~071.png   36 张  ← C段战场敌人原图（当前未被战场引用）
assets/card_icons/units/vis_player_036~071.png  42 张  ← 缴获卡面（当前被战场错误引用）
```

两者都在，问题纯粹是**引用错误**，不是缺图。`vis_enemy_036~071` 目前是"死文件"（仅缴获卡面被引用）。

---

## 五、FIXED_ENEMY_IDS 顺序（与 vis_enemy_036~071 严格对应）

定义位置：`data/enemy_unit_manifest.gd:57`

| 索引 | archetype_id | 战场图(应为) |
|------|--------------|-------------|
| 0 | enemy_ww1_infantry_basic | vis_enemy_036 |
| 1 | enemy_ww1_infantry_rifle | vis_enemy_037 |
| ... | ... | ... |
| 35 | boss_future_nexus | vis_enemy_071 |

与 `游戏单位清单_v3.md` C段表格完全一致。

**战场敌人原图计算公式**：
```gdscript
"vis_enemy_%03d" % (36 + FIXED_ENEMY_IDS.find(archetype_id))
```

## 六、修复方案

### 方案 A（推荐）：战场 sprite 用 vis_enemy，卡牌图标保持 vis_player

符合文档 C段（战场敌人 `vis_enemy`）+ C'/C''段（缴获卡面 `vis_player`）的分工。缴获卡面素材继续用于卡牌图标。

**实施步骤**：

**改动1** — `data/enemy_unit_manifest.gd`，在 `get_unit_icon_path_for_archetype`（约 :395）附近新增函数：

```gdscript
## 战场 sprite 专用：C段固定敌人用 vis_enemy（敌人原图），而非缴获卡面 vis_player。
## 仅供 resolve_card_icon_texture_path（战场贴图）使用；卡牌图标仍用 visual_id（vis_player）。
static func get_battlefield_icon_path_for_archetype(archetype_id: String) -> String:
	var aid := String(archetype_id).strip_edges()
	if aid.is_empty():
		return ""
	var fixed_idx: int = FIXED_ENEMY_IDS.find(aid)
	if fixed_idx < 0:
		return ""
	var rel := "vis_enemy_%03d" % (36 + fixed_idx)
	var full := "%s%s.png" % [UNITS_ICON_DIR, rel]
	return full if ResourceLoader.exists(full) else ""
```

**改动2** — `data/enemy_archetypes.gd`，`resolve_card_icon_texture_path`（:447）中，在 `explicit card_icon_path` 检查之后、`get_unit_icon_path_for_archetype` 之前插入：

```gdscript
	# 战场 sprite：C段固定敌人优先用 vis_enemy（敌人原图），而非缴获卡面 vis_player
	var battlefield_icon: String = EnemyUnitManifest.get_battlefield_icon_path_for_archetype(archetype_id)
	if not battlefield_icon.is_empty():
		return battlefield_icon
```

### 方案 B：战场敌人和卡牌图标都用 vis_enemy（统一用敌人原图）

改 `_visual_id_for_source_id`（:805）对 FIXED_ENEMY_IDS 返回 `"vis_enemy_%03d" % (36 + fixed_idx)`。

**副作用**：缴获卡牌图标也会显示敌人原图，`vis_player_036~071`（缴获卡面素材）变为冗余死文件。不推荐，除非确认缴获卡牌也要显示敌人原图。

---

## 七、验证方法

1. 运行游戏，进入有 C段敌人（一战/二战/冷战/现代/近未来）的关卡
2. 确认战场敌人显示 `vis_enemy_036~071`（敌人原图）
3. 打开背包/卡牌面板，确认缴获卡牌图标仍显示 `vis_player_036~071`（缴获卡面）
4. 可用临时脚本 `load()` 三个改动的场景验证编译（参考之前的 verify_panels.gd 模式）

---

## 八、注意事项

- **FORT_ENEMY_IDS（堡垒 E段）不受影响**：`_visual_id_for_source_id`（:792-795）对堡垒已正确返回 `vis_enemy_072~081`，战场图正确。
- **CAPTURED_ENEMY_IDS（C'段缴获）**：`_visual_id_for_source_id`（:797-799）返回 `vis_player_036~049`。这些是缴获卡面，正确。
- **`_sprite_resource_path_suspicious`**（enemy_unit.gd:801）：会忽略含 "background"/"bg_"/"backgrounds/" 的路径。`vis_enemy` 路径不含这些关键词，不会被误判。
- **POOL_ENEMY_IDS（D段）**：返回 `vis_pool_*`，无此问题。
- **FOE_PLATFORM/SPECIAL（A/B段）**：返回 `vis_player_001~028`/`030~035`，是玩家平台/精英图，无此问题。
