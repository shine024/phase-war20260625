## 2. 世界观与背景设定（当前版本）

### 2.1 核心世界观

- **时代跨度**：一战 → 二战 → 冷战 → 现代 → 近未来的架空战争史
- **叙事基调**：硬核军事装备感（坦克、装甲车、机枪、导弹、光束武器）与科幻元素（相位法则、高能平台）并存
- **玩家角色**：掌控"相位仪"的指挥官，通过远程投射构装单位作战
- **核心概念**："构装"技术改写战争——将战斗卡直接投射为自动化战争机器
- **科幻元素**：相位技术、法则系统（钢铁/烈焰/雷霆/虚空四大法则家族）、纳米材料、知识值

### 2.2 关键术语

| 术语 | 含义 |
|------|------|
| 相位仪 (Phase Instrument) | 玩家核心装备，4色槽位装配卡牌 |

| 相位场驱动器 | 玩家基地，被摧毁即失败 |
| 相位法则 (Phase Law) | 4大元素家族的战斗技能 |

| 纳米材料 | 基础货币，用于强化卡牌 |

## 3. 核心游戏循环

### 3.1 三层循环

```
┌─────────────────────────────────────────────────────────┐
│  30秒循环: 部署单位 → 自动战斗 → 施放法则                │
│  5分钟循环: 准备编队(相位仪) → 战斗 → 结算(1-3星)        │
│  长期循环: 击杀敌人获得卡牌 → 研究点升星 → 强化/改装/进化 │
└─────────────────────────────────────────────────────────┘
```

### 3.2 详细流程

1. **战前准备**：整理背包，将战斗卡/能量卡/法则卡装入相位仪
2. **进入战斗**：我方从左侧生成推进，敌方从右侧波次刷新
3. **自动战斗**：双方单位相向移动，进入射程后自动攻击
4. **胜负结算**：胜利获得资源/蓝图/星级评价(1-3★)；失败则返回准备
5. **长期推进**：100关 / 5时代（一战→近未来），逐步升级卡牌

### 3.3 战斗星级评价

| 星级 | 条件 |
|------|------|
| 1★ | 胜利即可 |
| 2★ | 单位存活率 ≥ 50% |
| 3★ | 存活率 ≥ 80% 且 用时 ≤ 预估时间×70% |

## 4. 时代与关卡结构

### 4.1 5个战争时代

| 时代 | 编号 | 关卡范围 | 掉落倍率 | 波次范围 | 每波敌人 | 波次间隔 | 难度系数 |
|------|------|----------|----------|----------|----------|----------|----------|
| 一战 WW1 | 0 | 1-20 | 0.85 | 3-5 | 1-2 | 14.0s | 0.80-1.18 |
| 二战 WW2 | 1 | 21-40 | 1.00 | 4-7 | 2-3 | 13.0s | 0.80-1.18 |
| 冷战 Cold War | 2 | 41-60 | 1.15 | 5-8 | 2-3 | 12.0s | 0.80-1.18 |
| 现代 Modern | 3 | 61-80 | 1.25 | 6-9 | 2-4 | 12.0s | 0.80-1.18 |
| 近未来 Near Future | 4 | 81-100 | 1.40 | 7-10 | 3-4 | 11.0s | 0.80-1.18 |

难度修正公式：`difficulty = 0.8 + (level_in_era - 1) × 0.02`

### 4.2 时代缩放公式（v3 重平衡）

**战斗卡 v3 缩放公式（来自 `data/battle_card_v3.gd`）：**

- **伤害**：`base_damage × era_damage_multiplier`
  - 缩放倍率：`1.00 / 1.20 / 1.40 / 1.65 / 1.90`（每级递增，曲线微降）
  - 近未来(era4)：×1.90（原×2.0）
- **射程**：`base_range × (1 + era × 0.10)`
  - 近未来(era4)：×1.4
- **生命值**：`base_hp × (1 + era × 0.15)`
  - 近未来(era4)：×1.6

**星级缩放公式：**
- **属性倍率**：`1.0 + (star - 1) × 0.08`（每星+8%）
- **品质加成**：rare/epic/legendary/mythic 额外每星+1%~4%

## 5. 卡牌体系 v3（三种卡统一模型）

### 5.1 v3 重构概述

**旧系统（已弃用）：**
- 平台卡 + 武器卡 = 战斗单位
- `CardType.PLATFORM` / `WEAPON` / `COMBINED`
- 需要从 `UnitStatsTable` 查表获取属性

**新系统（v3）：**
- **战斗卡**直接包含完整属性
- 武器作为战斗卡的内嵌属性（`weapon_label` 纯显示用）
- 100张敌人卡是数据基础，每张卡自带完整属性

### 5.2 三种卡类型

| 卡类型 | 编号 | 说明 | 核心字段 |
|--------|------|------|----------|
| 战斗卡 | 0 | 投射为战斗单位 | era, combat_kind, base_hp, base_damage, base_range, base_interval, base_speed, base_defense, weapon_label |
| 能量卡 | 1 | 战斗中能量回复/起始能量 | energy_grant |
| 法则卡 | 2 | 主动/被动战斗技能 | linked_law_id |

### 5.3 战斗卡完整字段列表

**通用字段（三种卡共有）：**
- `card_id`: 卡牌唯一标识
- `display_name`: 显示名称
- `description`: 详细规则文本
- `card_type`: 卡类型（0=战斗, 1=能量, 2=法则）
- `energy_cost`: 部署能量消耗
- `rarity`: 品质（common/uncommon/rare/legendary）
- `star_level`: 星级（1-9）

**战斗卡专用字段（card_type == 0）：**
- `era`: 时代（0-4，WW1到近未来）
- `combat_kind`: 战斗定位（0=轻装/1=装甲/2=炮台/3=支援）
- `base_hp`: 基础生命值
- `base_damage`: 基础攻击伤害
- `base_range`: 基础攻击射程（像素）
- `base_interval`: 基础攻击间隔（秒）
- `base_speed`: 基础移动速度（0=固定）
- `base_defense`: 基础防御值
- `weapon_label`: 武器外观标签（如"步枪"、"机枪"、"迫击炮"）

**能量卡专用字段（card_type == 1）：**
- `energy_grant`: 使用后获得的能量

**法则卡专用字段（card_type == 2）：**
- `linked_law_id`: 对应的法则ID

### 5.4 100张敌人卡结构

**数据来源：** `data/enemy_unit_manifest.gd`

**分段说明：**

| 段落 | 数量 | 说明 | 卡ID示例 |
|------|------|------|----------|
| A段 | 29张 | 原我方平台 → 敌人 archetype（前缀 foe_） | platform_ww1_light, platform_ww2_medium... |
| B段 | 6张 | 原精英掉落平台 | bulwark, titan_mk2, storm_rider... |
| C段 | 36张 | 固定敌人（与 enemy_archetypes.json 一致） | enemy_ww1_infantry_basic, boss_ww2_kingtiger... |
| D段 | 29张 | 补充敌人池（foe_pool_001~029） | foe_pool_001, foe_pool_002... |

**属性定义方式：**
- A/B段：通过 `_get_foe_stats()` 函数直接定义完整属性
- C段：从 `enemy_archetypes.gd` 读取
- D段：补充敌人池，用于随机生成

**示例（A段 - platform_ww1_light）：**
```gdscript
{
  "kind": 0,        # combat_kind: 0=轻装
  "hp": 65.0,       # base_hp
  "dmg": 8.0,       # base_damage
  "rng": 95.0,      # base_range
  "ivl": 0.38,      # base_interval
  "spd": 115.0,     # base_speed
  "def": 5.0,       # base_defense
  "weapon": "冲锋枪" # weapon_label
}
```

### 5.5 战斗卡战斗定位（combat_kind）

| 编号 | 定位 | 特点 | 典型单位 |
|------|------|------|----------|
| 0 | 轻装 | 高速度、低耐久、侦察骚扰 | 侦察车、步兵班、突袭车 |
| 1 | 装甲 | 中高耐久、中速、主力输出 | 坦克、机甲、重装单位 |
| 2 | 炮台 | 固定不动、高耐久、远程火力 | 要塞炮、雷达站、自行火炮 |
| 3 | 支援 | 战场维修、载机母舰、辅助功能 | 救护车、步战车、母舰 |

## 15. 战斗系统核心公式

### 15.1 伤害计算

```
final_damage = base_damage × era_damage_multiplier × star_stat_multiplier × crit_multiplier × (1 - damage_reduction × (1 - armor_penetration))
splash = final_damage × splash_damage
shield_cap = 2 × max_hp
```

- **暴击**：2x倍率
- **护盾上限**：2倍最大生命值
- **时代缩放**：见第4.2节 v3 公式
- **星级缩放**：见第10.2节

### 15.2 部署公式

```
deploy_energy_cost = card.energy_cost
deployment_time = deploy_energy_cost / energy_output_rate
```

### 15.3 战力评分

```
power_score = (80 + star×28 + mod_count×22) × rarity_mul × (1 + inherit_bonus)
```

### 15.4 存活率与星级

```
survival_rate = (max_deployed - units_lost) / max_deployed
stars = 1 (胜利)
stars = 2 if survival_rate ≥ 0.5
stars = 3 if survival_rate ≥ 0.8 AND battle_time ≤ estimated_time × 0.7
```
