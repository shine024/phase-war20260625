## 一、核心设计原则（已确认）

| 原则             | 说明                               |
| ---------------- | ---------------------------------- |
| **自动对战**     | 玩家布阵，系统自动战斗，战中无操作 |
| **策略在战前**   | 所有决策在编队界面完成             |
| **克制决定胜负** | 攻击/防御的多维克制是核心          |

## 六、成长系统

### 6.1 进化规则（重要）

> **核心**：进化不是“E0→E1”阶段成长，而是**变成另一张更高级的卡**（转职/进化）

| 规则             | 说明                                                 |
| ---------------- | ---------------------------------------------------- |
| 战力达标         | 基础单位通过强化+改造，战力达到目标单位的基础战力    |
| 情报100%         | 获得目标单位的完整情报（战斗中遇到、击败、侦察获得） |
| 进化选择         | 基础单位 → 变成目标单位（同系列，如步兵→步兵）       |
| 改造继承         | 旧改造保留（按比例），可被新单位的改造替换           |
| 新单位可继续成长 | 强化、改造、再进化                                   |

### 6.2 情报手册系统

| 行为              | 情报获取量 |
| ----------------- | ---------- |
| 首次遭遇该单位    | +20%       |
| 击败普通单位      | +5%~10%    |
| 击败精英/BOSS     | +15%~25%   |
| 使用侦察单位/法则 | +5%~15%    |
| 重复卡分解        | +10%       |

## 💰 经济平衡

| 单位       | 进化前战力 | 目标战力  | 需要额外培养 | 消耗资源  |
| ---------- | ---------- | --------- | ------------ | --------- |
| MP18(基础) | 50         | 120(海军) | +70战力      | ~8000纳米 |
| MP18(满培) | 125        | 120(海军) | 0            | 只需情报  |
| MP18(满培) | 125        | 160(特种) | +35战力      | ~4000纳米 |

**玩家策略**：
- 直接进化：基础MP18需要大量培养
- 先培养再进化：节省进化成本
- 满培后跳级进化：省中间环节

## 💎 总结

| 你的设计       | 效果                       |
| -------------- | -------------------------- |
| 战力是进化资格 | 必须培养才能进化，不能白嫖 |
| 情报是进化钥匙 | 必须打过/研究过敌人        |
| 进化后改造继承 | 之前的培养不浪费           |
| 改造可替换     | 新单位有新改造选项         |
| 多分支选择     | 同一基础单位可走不同路线   |

**一句话**：培养→战力达标→情报解锁→进化成新单位→继续培养→再进化。

### 1.3 选敌逻辑详细规则

#### 直射(DIRECT)

```
目标选择优先级：
1. 距离最近的敌人（欧几里得距离）
2. 同距离时，选择当前生命值最低的敌人
3. 同距离同生命时，选择部署时间最早的

特殊：射程内无敌人时，向敌方基地方向移动
```

#### 曲射(INDIRECT)

```
目标选择优先级：
1. 优先选择【被本单位的攻击类型克制】的敌人
   - 如果 attack_armor 高 → 优先选装甲类敌人
   - 如果 attack_light 高 → 优先选轻装类敌人
   - 如果 attack_air 高 → 优先选空中类敌人
2. 无克制目标时，选择距离最近的敌人
3. 同距离时，选择生命值最低的

特殊：不移动（部署速度通常为0或极低），全图攻击
```

#### 空射(AERIAL)

```
目标选择优先级：
1. 优先选择【空中单位】（空对空）
2. 无空中目标时，选择被地面攻击类型克制的敌人
3. 无克制目标时，选择距离最近的

特殊：飞行单位，可被防空单位攻击
```

## 二、射程衰减系统

### 2.1 衰减公式

```gdscript
# 仅直射武器有衰减
# 伤害衰减率 = 1 - (距离 / 最大射程) × 衰减系数

func calculate_damage_attenuation(distance, max_range, attenuation_factor = 0.5):
    if distance <= 0:
        return 1.0
    if distance >= max_range:
        return 1.0 - attenuation_factor  # 最远距离衰减 attenuation_factor%
    
    # 线性衰减
    var t = distance / max_range
    return 1.0 - (t * attenuation_factor)
```

### 2.2 衰减系数表

| 武器类型    | 衰减系数 | 最远衰减 | 示例         |

| 机枪/MG     | 0.5      | -50%     | MG42、M60    |
| 坦克炮/TANK | 0.3      | -30%     | 虎王、M1A2   |
| 反坦克炮/AT | 0.2      | -20%     | 铁拳、标枪   |
| 狙击/SNIPER | 0.1      | -10%     | 特殊单位     |

## 四、完整战斗流程

### 4.1 选敌→伤害流程

```
每帧/每次攻击判定：

1. 获取攻击者的武器类型（DIRECT/INDIRECT/AERIAL）

2. 选敌：
   - DIRECT: 遍历所有敌方单位，计算距离，选最近
   - INDIRECT/AERIAL: 遍历所有敌方单位，按优先级选
     优先级 = 是否被克制(权重2) + 距离(权重1)

3. 检查射程：
   - DIRECT: 距离 > range ? 无法攻击
   - INDIRECT/AERIAL: 无射程限制

4. 计算伤害：
   var damage = get_attack_value(attacker, target)
   # 根据目标类型选择 attack_light/armor/air

5. 应用衰减（仅DIRECT）：
   var attenuation = calculate_attenuation(distance, range, weapon_type)
   damage = damage × attenuation

6. 应用防御减免：
   var defense = get_defense_value(target, attacker)
   # 根据攻击者类型选择 defense_light/armor/air
   damage = damage × (100 / (100 + defense))

7. 造成伤害
```

### 4.2 部署速度含义

| 部署速度 | 含义     | 到达战场时间（基准） | 典型单位           |
| -------- | -------- | -------------------- | ------------------ |
| 0        | 瞬间部署 | 0秒                  | 固定炮台、要塞     |
| 1        | 极慢     | 10秒                 | 重型坦克、重型火炮 |
| 2        | 慢       | 8秒                  | 虎王、巨神机甲     |
| 3        | 中慢     | 6秒                  | 主战坦克、中型坦克 |
| 4        | 中速     | 4秒                  | 步兵、步战车       |
| 5        | 快速     | 2.5秒                | 轻型坦克、反坦克车 |
| 6        | 极快     | 1.5秒                | 侦察车、无人机     |
| 7        | 瞬间进场 | 0.5秒                | 空降兵、空投单位   |

**公式**：`到达时间(秒) = (8 - deploy_speed) × 1.5`（deploy_speed≤7时）
