## 1. 单位情报系统

### 概述

情报系统（v6.0 情报驱动进度系统）将原本单一进度条扩展为 **4 维情报体系**，包含揭秘事件、敌方起源改造（EOM）和隐藏进化分支。

```
BattleDamageSystem (generate_battle_completion_drops)
  │
  ▼
IntelDiscoveryManager.generate_battle_intel_harvest()
  │  ┌─ register_first_encounter()
  │  ├─ register_defeat()
  │  ├─ register_recon()
  │  ├─ _check_reveals()      → IntelRevealEvents
  │  ├─ _check_eom_drops()
  │  └─ _roll_intel_item_drops() → IntelManualItems
  ▼
IntelManual (核心数据模型，4 维存储)
  │
  ├── IntelManualUI (缓存查询层)
  ├── IntelItemBag (消耗品背包)
  ├── IntelEvolutionManager (隐藏分支发现)
  └── IntelDimensions (权重/分配逻辑)
```

## 2. 强化与进化系统

### 2.1 强化系统 (Card Enhancement)

#### 概述

卡牌强化使用纳米材料将单位从 Lv1 提升至 Lv10，每次强化获得属性加成。成功率 100%（v5.0 设计），消耗强化手册 + 纳米材料。

#### 核心文件

| 文件 | 类名 | 行数 | 职责 |
|------|------|------|------|
| `managers/card_enhancement_manager.gd` | — | 114 | 强化核心逻辑 |
| `scenes/ui/card_enhancement_panel.gd` | `CardEnhancementPanel` | 296 | 强化+升星+改装+进化整合 UI |
| `scenes/ui/enhancement_animation.gd` | `EnhancementAnimation` | 109 | 强化动画 |
| `design/gdd/enhancement-system.md` | — | ~80 | 强化系统 GDD |

#### 强化等级与属性加成

| 等级 | 费用倍率 | 属性加成 | 战力倍率 |
|------|----------|----------|----------|
| 1→2 | 0.5x | +5% | 1.05 |
| 2→3 | 1.0x | +10% | 1.10 |
| 3→4 | 1.5x | +15% | 1.15 |
| 4→5 | 2.0x | +20% | 1.20 |
| 5→6 | 2.5x | +25% | 1.25 |
| 6→7 | 3.0x | +30% | 1.30 |
| 7→8 | 3.5x | +35% | 1.35 |
| 8→9 | 4.5x | +50% | 1.50 |
| 9→10 | 6.0x | +60% | 1.60 |

#### 计算公式

```gdscript
纳米消耗 = card.power × level_cost_multiplier
战力倍率 = 1.0 + level * 0.05  (Lv1-8)
          1.50 (Lv9) / 1.60 (Lv10)
```

#### 按卡牌类型属性加成

| 卡牌类型 | 加成属性 |
|----------|----------|
| COMBAT_UNIT | hp_bonus = bonus, weight_bonus = bonus * 0.2 |
| ENERGY | energy_grant_bonus = bonus |
| LAW | effect_bonus = bonus |
| default | damage_bonus = bonus |

强化后能量消耗也会微增：`energy_cost × (1.0 + bonus × 0.5)`

#### 进化执行流程

```gdscript
can_evolve_blueprint(card_id, target_card_id):
  1. 蓝图已解锁? → bpm_ref.is_blueprint_unlocked(card_id)
  2. 目标有效? → DefaultCards 或 PhaseLaws 中存在
  3. 目标在进化路径? → 匹配 evolution_1 / faction_branch / intel_branch
  4. combat_kind 匹配? → 需相同 (intel_branch 可跨兵种)
  5. 战力达标? → 当前战力 >= 目标基础战力
  6. 情报 100%? → target_intel_progress >= 1.0
  7. 强化等级 → E1: ≥5, E2: ≥8
  8. 改装数量 → E1: ≥2, E2: ≥5
  9. 起源改装 → E2: 需至少 1 个 EOM

evolve_blueprint(card_id, target_card_id):
  继承倍率 = max(旧继承倍率, 0.30)
  继承加成 = clamp(旧加成 + 0.30, 0.0, 0.9)  # 上限 90%
  星级 = max(源星级, 目标当前星级)
  HP底线 = old_HP × 1.10  (如果 old_HP > 0)
  改装 = 完全继承 (源→目标, 源清空)
  强化等级 = 重置为 1
  蓝图解锁 = 自动解锁目标
```

#### 冲突组规则

按 MOD 类型分冲突组，同类型替换，不同类型追加。

- 无冲突且 < 9 个 → 追加（计数+1）
- 无冲突但已达 9 个 → 拒绝

#### 核心消耗公式

```gdscript
部署能量 = max(1, 平台费用 + Σ武器费用)
法则环境匹配度 = 0.5 + (匹配维度数 / 总维度数) × 0.5  // 范围 50%-100%
强化费用 = card.power × level_cost_multiplier
升星战力倍率 = 稀有度倍率 × (1 + (星级-1) × 0.08)
伤害计算 = base_damage × (暴击 ? 2x : 1x)
伤害减免 = 最终伤害 × (1 - 护甲 × (1 - 穿透率))
```
