## 修订说明（相对初版 v3.0 文档）

1. **费用(COST) 与类型基表（§1.2）**：`PlatformType` 表中的「费用」为 **多数卡在时代 0 下的参考部署费**，不是硬约束。单张战斗卡可在各时代表中单独标注（例如 `cold_hound` 4 费、`ww2_titan` 6 费），以实现平衡，**不视为笔误**。
2. **稀有度与升星（§3.2）**：补全 **Mythic** 列；Mythic 的每段升星研究点 = 同段 Common 消耗 × **8.0**（与 §1.4 升星消耗系数一致），合计 **26,480** 研究点。
3. **强化失败（§4.1）**：明确为 **仅损失本次投入的纳米**，**强化等级不变**（不掉级、不损毁卡牌）。表中「累计纳米」为成功路径；失败时按成功率另行统计期望。
4. **乘区叠加上限（平衡约束）**：继承加成公式与单次/总上限仍以 §6.4 为准；**建议实现侧**对「终局特性 + 改装百分比 + 稀有度/星级」叠加设置软顶或 PvE 难度随动，避免后期一刀清屏（详见 SCHEMA 文档中的运行时建议）。

### 1.3 时代缩放公式

```
时代伤害倍率 = 1 + era × 0.25
时代射程倍率 = 1 + era × 0.10
时代生命倍率 = 1 + era × 0.15
```

实现：`BattleCardV3.era_damage_multiplier` / `era_range_multiplier` / `era_hp_multiplier`（`era` 为 0~4）。  
战斗派生单位时，`UnitStatsTable.build_stats` / `build_multi_stats` 在 `era >= 0` 下对武器伤害/射程使用前两式，对 **`max_hp`（平台基础生命）** 使用 `era_hp_multiplier`（此前仅武器随时代缩放，已与 v3 对齐）。

## 六、进化系统

### 6.1–6.2 进化要求

与初版一致（E1：4★、+6、满改装、研究点 320、纳米 1500、合金 50…；E2：7★、+8…）。

### 6.3 进化路线与准入

含「**任意平台 / 任意单位**」的路线，在数据层使用 **`evolution_entry_tags` 或允许的 `card_id` 白名单** 声明准入（见 SCHEMA）。路线 C 起点「WW1 阵地」指 **`ww1_fortress`** 或策划指定的同 tag 要塞类卡。

### 6.4 继承机制

```
继承加成 = (星级 - 1)×4% + 强化等级×1%
单次最大 = 40%
多次进化可叠加，上限 = 100%
```
